# BuoyCalc Windows — CI status policy

## Причина

При прямых коммитах в `main`, созданных через GitHub connector, в текущем инструменте проверки может возвращаться пустой список `workflow_runs: []`.

При PR-коммитах workflow виден через GitHub Actions API. Пример: открытый PR #1 показывает `.NET Build` со статусом `completed / success`.

## Рабочее правило

Для дальнейших инженерных шагов BuoyCalc Windows использовать PR-ветку:

```text
создать ветку от main
внести изменение в ветке
открыть pull request в main
проверять .NET Build по head SHA pull request
после успешной проверки принимать решение о merge
```

## Что это меняет

Это не меняет расчётную физику, solver, 2D, PDF или модель данных.

Изменяется только процесс контроля:

```text
статус CI проверяется по PR head SHA,
а не по прямому commit SHA в main.
```

## Что нельзя делать

```text
не продолжать серию инженерных изменений прямыми коммитами в main без видимой CI-проверки
не считать workflow_runs: [] успешной сборкой
не смешивать исправление процесса CI с изменениями расчётной физики
```
