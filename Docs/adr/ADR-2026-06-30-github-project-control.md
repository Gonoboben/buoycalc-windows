# ADR-2026-06-30: Ведение BuoyCalc Windows через GitHub с архитектурной хронологией

Статус: принято к рассмотрению через PR

Дата: 2026-06-30

## Контекст

`BuoyCalc Windows` развивается как отдельная инженерная расчётная система.

Важно: `BuoyCalc Windows` и `PlataVM / плата питания` — абсолютно разные проекты. Они не должны иметь общую архитектурную базу, общую хронологию или общие проектные решения.

В BuoyCalc Windows есть риск, что новые изменения начнут незаметно менять архитектуру: solver, структуру последовательности элементов, расчётную модель, PDF/2D-представление или пользовательскую терминологию.

Такой дрейф опасен: расчётная система может стать нестабильной, а результаты — недостоверными.

## Решение

Вести `BuoyCalc Windows` через GitHub с обязательными контрольными документами:

1. `Docs/PROJECT_CONTROL.md` — правила ведения проекта BuoyCalc Windows.
2. `Docs/ARCHITECTURE_BASELINE.md` — зафиксированная архитектурная база BuoyCalc Windows.
3. `Docs/CHRONOLOGY.md` — последовательная хронология изменений BuoyCalc Windows.
4. `Docs/adr/` — каталог архитектурных решений BuoyCalc Windows.

Все изменения, влияющие на архитектуру BuoyCalc Windows, выполняются только через отдельную ветку и PR.

## Последствия

Положительные:

1. Появляется единая точка сверки перед каждым новым шагом по BuoyCalc Windows.
2. Снижается риск случайной смены архитектуры расчётного приложения.
3. Можно восстановить логику: почему было принято то или иное решение.
4. Можно отделять расчётные изменения от косметических.
5. BuoyCalc Windows не смешивается с PlataVM/платой питания.

Ограничения:

1. Документацию нужно обновлять после каждого значимого изменения.
2. Нельзя быстро «переделать как кажется лучше», если это ломает зафиксированную модель.
3. Новые идеи сначала проходят через ADR или отдельный план внедрения.
4. Если задача относится к PlataVM, её нельзя вести в документах BuoyCalc Windows.

## Проверка

Для каждого будущего PR проверяется:

1. обновлена ли хронология BuoyCalc Windows;
2. не нарушена ли архитектурная база BuoyCalc Windows;
3. не внесены ли решения из PlataVM/PDU в BuoyCalc Windows;
4. не изменилась ли физика расчёта без явного указания;
5. пройден ли CI, если затронут код.
