# BuoyCalc Windows v0.30

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.30 добавляет геометрический мост между итерационной формой X/Z и будущим полным solver сил. Теперь программа не только строит узлы формы, но и раскладывает форму на проекции сегментов: `dX`, `dZ`, фактическую длину по координатам, невязку длины и угол сегмента.

Это ещё не полный нелинейный solver равновесия, но это необходимый промежуточный слой:

```text
форма X/Z → проекции сегментов → будущий пересчёт локальных сил → новые натяжения → новая форма
```

## Что изменилось в v0.30

1. Добавлен сервис `MooringShapeProjection`.
2. Добавлены строки проекций формы по каждому сегменту:
   - `dX, м`;
   - `dZ, м`;
   - `L по X/Z, м`;
   - `Невязка L, м`;
   - `Угол, °`;
   - `T, кН`;
   - `Статус`.
3. В текстовом отчёте появился раздел:

```text
## Проекции формы X/Z по сегментам
```

4. В итогах отчёта появились сводные строки:
   - `ΣdX формы`;
   - `ΣdZ формы`;
   - `Статус проекций формы`.
5. Версия приложения обновлена:

```text
v0.30 · shape projection bridge
```

## Что именно проверяет v0.30

v0.30 проверяет внутреннюю геометрию построенной формы:

```text
ΣdX = Xякоря - Xбуя
ΣdZ = Zякоря - Zбуя
ΣLпо координатам ≈ ΣLсегментов
```

Для каждого сегмента считается:

```text
dX = Xконец - Xначало
dZ = Zконец - Zначало
Lxz = sqrt(dX² + dZ²)
угол = atan(|dX| / |dZ|)
```

Если длина по координатам не совпадает с длиной сегмента, строка получает `WARNING`. Это важно, потому что дальнейший solver не должен пересчитывать силы по геометрии, которая сама с собой не согласована.

## Что осталось от v0.29

v0.29 добавил первый итерационный solver формы постановки. `MooringShapeSolver` больше не возвращает постоянное `Converged = false`; форма имеет реальный критерий сходимости по якорной глубине:

```text
|Zякоря - Depth| ≤ 0,01 м
```

В инженерной диагностике остаются строки:

```text
Итерации solver формы
Невязка сходимости формы
Статус сходимости формы
```

## Что v0.30 ещё не решает

v0.30 пока не пересчитывает силы по новой ориентации сегментов. Полная задача ещё не замкнута:

```text
форма линии ⇄ локальные силы ⇄ натяжения ⇄ углы ⇄ новая форма ⇄ реакции
```

Поэтому строка:

```text
Полный баланс всей постановки
```

по-прежнему должна оставаться `WARNING`.

## Что осталось от v0.28.1

В текстовом отчёте остаётся раздел:

```text
## Векторная ведомость сил постановки
```

Он показывает вклад каждого силового члена в `ΣFx` и `ΣFz`:

```text
буй
соединители
приборы
линия
якорь
волновая добавка, если она есть
```

## Что осталось от v0.27.1

Строки внутреннего контроля линии остаются информационными:

```text
ΣFx линии
ΣFz линии
Контроль накопления ΣFx линии
Контроль накопления ΣFz линии
```

Они не являются полной проверкой равновесия всей системы. Это внутренний контроль накопления сил текущей сегментной модели линии.

## Как проверить

1. Запустить программу.
2. Убедиться, что в правом верхнем углу и в окнах написано:

```text
v0.30 · shape projection bridge
```

3. Выполнить расчёт.
4. Открыть `Отчёт текстом...`.
5. Найти раздел `Итоги` и проверить строки:
   - `ΣdX формы`;
   - `ΣdZ формы`;
   - `Статус проекций формы`.
6. Найти раздел:

```text
## Проекции формы X/Z по сегментам
```

7. Проверить, что таблица содержит `dX`, `dZ`, `L по X/Z`, `Невязка L`, `Угол`, `T`.
8. Проверить, что `Полный баланс всей постановки` пока остаётся `WARNING`.

## Следующий шаг

Рекомендуемый v0.31:

1. использовать проекции формы для пересчёта локальной силы течения на сегмент;
2. отделить силу на сегмент от старой табличной оценки;
3. пересчитать натяжения по обновлённым локальным силам;
4. выполнить итерацию:

```text
форма → проекции → локальные силы → натяжения → углы → новая форма
```

5. после этого начать снижать предупреждение `Полный баланс всей постановки` только тогда, когда будет реальный критерий силовой сходимости.
