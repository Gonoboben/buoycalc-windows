# BuoyCalc Windows v0.38

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.38 добавляет отдельную PDF-страницу сравнения основной и альтернативной формы X/Z. Это визуальный слой поверх уже рассчитанных данных: PDF не придумывает геометрию, а отображает точки из `MooringShapeStore` и `MooringAlternativeShapeStore`.

Основной solver пока не заменяется. Альтернативная форма остаётся сравнительной.

## Что изменилось в v0.38

1. В `PdfReportBuilder` добавлена страница:

```text
Сравнение форм X/Z
```

2. Страница появляется только если доступны обе формы:
   - основная форма `MooringShapeStore.Current`;
   - альтернативная форма `MooringAlternativeShapeStore.Current`.
3. На странице рисуются:
   - основная форма X/Z;
   - альтернативная форма X/Z;
   - дискретные X/Z-точки приборов, соединителей и вертлюгов;
   - основной и альтернативный снос;
   - статус альтернативной формы `OK` или `WARNING`.
4. Масштаб единый для горизонтали и глубины:

```text
X = Z
```

5. Если альтернативная форма имеет предупреждение по `angleScale`, PDF показывает `альтернативная форма: WARNING`.
6. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.38 - PDF comparison
```

## Что важно методически

PDF-страница сравнения форм не читает координаты из Markdown-таблиц отчёта. Она использует расчётные объекты:

```text
MooringShapeStore.Current
MooringAlternativeShapeStore.Current
```

Это продолжает правило проекта:

```text
визуализация отображает модель, но не создаёт физику
```

## Что v0.38 ещё не делает

v0.38 не заменяет основной solver и не объявляет альтернативную форму финальным решением. Это страница сравнения для проверки PDF и разных расчётных параметров.

Полный баланс всей постановки остаётся `WARNING`.

## Что осталось от v0.37.1

GitHub Actions CI остаётся включённым:

```text
.github/workflows/dotnet-build.yml
```

После каждого push в `main` должен запускаться `.NET Build`.

## Как обновиться локально

```text
Git -> Pull
Build -> Clean Solution
Build -> Rebuild Solution
```

## Как проверить

1. Выполнить расчёт.
2. Открыть `2D-схема` и проверить основную/альтернативную форму.
3. Экспортировать PDF.
4. В PDF должна появиться отдельная страница:

```text
Сравнение форм X/Z
```

5. На странице должны быть:
   - основная форма;
   - альтернативная форма;
   - дискретные точки;
   - основной снос;
   - альтернативный снос;
   - статус альтернативной формы.
6. Проверить GitHub Actions:

```text
Actions -> .NET Build
```

## Следующий шаг

После того как пользователь пришлёт несколько PDF-отчётов с разными параметрами, сравнить:

1. снос основной формы;
2. снос альтернативной формы;
3. статус альтернативной формы;
4. влияние дискретных элементов на натяжения;
5. читаемость PDF-страницы сравнения.

После этой проверки можно переходить к итерационному solver:

```text
форма -> силы -> натяжения -> дискретные узлы -> новая форма -> сходимость
```
