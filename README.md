# BuoyCalc Windows v0.38.4

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.38.4 исправляет основную PDF-страницу `Схема постановки`. Теперь она использует расчётную форму из `MooringShapeStore.Current`, если форма доступна, а не рисует упрощённую диагональную схему с буем у поверхности.

Это важно для коротких линий и погружённого буя: PDF должен показывать фактическую расчётную геометрию, а не визуальный fallback.

## Что изменилось в v0.38.4

1. `PdfReportBuilder` теперь получает `mainShape` до построения страницы схемы:

```text
MooringShapeStore.Current
```

2. Если основная форма есть и содержит минимум два узла, страница `Схема постановки` вызывает:

```text
MainShapeDiagram(mainShape, sequence)
```

3. Упрощённая схема переименована в:

```text
FallbackMooringDiagram
```

и используется только если расчётная X/Z-форма недоступна.

4. Новая основная PDF-схема показывает:
   - поверхность воды;
   - дно;
   - основную форму X/Z;
   - фактический верхний узел линии;
   - якорь;
   - состояние буя;
   - статус сходимости формы;
   - расчётный снос X/Z.

5. Масштаб схемы остаётся единым:

```text
X = Z
```

6. Если буй под водой или форма не сошлась, на схеме выводится предупреждающий текст:

```text
форма: WARNING / not converged
```

7. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.38.4 - PDF solver diagram
```

## Что это исправляет

В коротколинейном варианте 80 м линии на 100 м глубины первая PDF-схема раньше могла выглядеть как обычная поверхностная постановка: буй рисовался у поверхности. После v0.38.4 эта страница должна показывать буй ниже поверхности, потому что берёт координаты из solver.

## Что не изменилось

1. Волновая нагрузка не отключается.
2. Логика короткой линии из v0.38.2 остаётся.
3. CI status bridge из v0.38.3 остаётся.
4. Страница `Сравнение форм X/Z` остаётся отдельной страницей PDF.

## Как обновиться локально

```text
Git -> Pull
Build -> Clean Solution
Build -> Rebuild Solution
```

## Как проверить

1. Пересчитать вариант с линией 80 м на глубине 100 м.
2. Экспортировать PDF.
3. На странице `Схема постановки` буй должен быть показан ниже поверхности, а не на поверхности.
4. На этой же странице должна быть подпись:

```text
форма: WARNING / not converged
```

5. На странице `Сравнение форм X/Z` альтернативная форма также должна остаться `WARNING`.
6. Проверить CI status:

```text
BuoyCalc Windows Build: success
```

## Следующий шаг

После проверки PDF v0.38.4 можно продолжить подготовку итерационного solver:

```text
форма -> силы -> натяжения -> дискретные узлы -> новая форма -> сходимость
```
