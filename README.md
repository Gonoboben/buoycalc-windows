# BuoyCalc Windows v0.27.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.27.1 выполняет методологическую чистку силовой диагностики v0.27. Строки с нулевыми значениями теперь называются не «невязками», а внутренним контролем накопления `ΣFx` / `ΣFz` линии и имеют статус `INFO`.

## Что изменилось в v0.27.1

1. строка `Невязка ΣFx сегментной суммы` переименована в `Контроль накопления ΣFx линии`;
2. строка `Невязка ΣFz сегментной суммы` переименована в `Контроль накопления ΣFz линии`;
3. обе строки переведены в статус `INFO`;
4. строка `Полный баланс всей постановки` оставлена как `WARNING`;
5. README явно поясняет, что текущий контроль накопления сил не является полной проверкой равновесия всей буйковой постановки.

## Что осталось от v0.27

v0.27 добавил силовую диагностику текущей сегментной модели линии. Отчёт показывает не только геометрические проверки, но и внутренний контроль сумм сил `ΣFx` / `ΣFz` по линии.

В диагностике остаются строки:

```text
ΣFx линии
ΣFz линии
Контроль накопления ΣFx линии
Контроль накопления ΣFz линии
Полный баланс всей постановки
```

## Что именно считается сейчас

Текущая силовая диагностика проверяет внутреннюю согласованность накопления сил внутри сегментной модели линии:

```text
ΣFx линии = сумма горизонтальных сил сопротивления по сегментам линии
ΣFz линии = сумма вертикальных весовых сил по сегментам линии
Ttop_x сейчас берётся из той же накопленной сегментной суммы
Ttop_z сейчас берётся из той же накопленной сегментной суммы
```

Поэтому контрольные значения:

```text
ControlFx = |ΣFx - Ttop_x|
ControlFz = |ΣFz - Ttop_z|
```

в текущей реализации закономерно равны нулю. Это не независимая проверка физического равновесия, а внутренний контроль того, что текущая сегментная модель линии согласованно передаёт накопленные силы в диагностический отчёт.

## Важное ограничение

Это ещё не полный баланс всей системы:

```text
буй + соединители + приборы + линия + якорь + реакции
```

Пока полный баланс специально выводится как предупреждение:

```text
Полный баланс всей постановки | ещё не решается | WARNING
```

Это честно: текущая модель уже показывает накопленные силы линии, но общий итерационный solver равновесия ещё не включён.

## Что осталось от v0.26

В отчёте остаётся раздел:

```text
## Контроль инженерной модели
```

Он включает геометрию, запасы, состояние буя, силовые информационные строки и предупреждение о неполном балансе постановки.

## Что осталось от v0.25.1

Основной 2D-вид использует равный масштаб:

```text
xScale = zScale
```

То есть координаты отображаются без искусственного увеличения по горизонтали.

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Открыть `Отчёт текстом...`.
4. Найти раздел `Контроль инженерной модели`.
5. Проверить, что появились строки:
   - `ΣFx линии`;
   - `ΣFz линии`;
   - `Контроль накопления ΣFx линии`;
   - `Контроль накопления ΣFz линии`;
   - `Полный баланс всей постановки`.
6. Проверить, что строки `Контроль накопления ΣFx линии` и `Контроль накопления ΣFz линии` имеют статус `INFO`.
7. Проверить, что полный баланс всей постановки пока имеет статус `WARNING`.
