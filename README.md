# BuoyCalc Windows v0.35.1

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.35.1 — методическая чистка после проверки PDF-отчёта v0.35. Физика solver-а не объявляется завершённой. Цель шага — не давать альтернативной форме статус `OK`, если она геометрически замыкается только за счёт чрезмерного масштабирования углов.

## Что исправлено в v0.35.1

1. Для альтернативной формы X/Z добавлен контроль масштаба углов.
2. Альтернативная форма получает `OK` только если одновременно:

```text
|Zякоря - Depth| <= 0.01 м
angleScale <= 2.0
```

3. Если `angleScale > 2.0`, отчёт выводит `WARNING`, даже если глубина якоря формально закрыта.
4. Методическое примечание альтернативной формы теперь прямо пишет, что большое масштабирование углов означает сравнительную, а не физически сошедшуюся форму.
5. PDF-отчёт теперь выводит версию расчётной модели на первой странице.
6. PDF footer теперь содержит версию приложения.
7. В PDF-таблице элементов уточнены заголовки:

```text
Вес -> Вес в воде
Сила -> Fx, Н
```

8. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.35.1 - alt shape warning
```

## Почему это важно

В проверенном отчёте v0.35 альтернативная форма имела:

```text
scale = 21.4219
статус = OK
```

Геометрически форма действительно попадала в проектную глубину, но такое масштабирование показывает, что углы из силовой ведомости и геометрия линии пока не согласованы как полноценный нелинейный solver. Поэтому статус `OK` был методически слишком сильным.

После v0.35.1 такой случай должен отображаться как:

```text
статус = WARNING
```

с пояснением, что форма полезна для сравнения, но не должна трактоваться как физически сошедшийся solver.

## Что осталось от v0.35

В отчёте остаётся раздел:

```text
## Альтернативная форма X/Z с дискретными нагрузками
```

Он по-прежнему сравнивает:

```text
X основной формы
Z основной формы
X альтернативной формы
Z альтернативной формы
ΔX
ΔZ
снос основной формы
снос альтернативной формы
max Δузла
```

## Что v0.35.1 ещё не делает

v0.35.1 не вставляет соединители и приборы как отдельные X/Z-узлы. Они продолжают влиять на альтернативную форму через натяжения и углы, но ещё не появляются как отдельные точки на линии.

Полный баланс всей постановки остаётся `WARNING`.

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...`.
3. В `Итоги` проверить строку:

```text
Статус альтернативной формы
```

4. Если `scale` альтернативной формы больше 2, статус должен быть `WARNING`.
5. Экспортировать PDF.
6. На первой странице проверить строку:

```text
Версия расчётной модели: v0.35.1 - alt shape warning
```

7. В PDF-таблице элементов проверить заголовки:

```text
Вес в воде
Fx, Н
```
