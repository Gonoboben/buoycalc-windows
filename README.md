# BuoyCalc Windows v0.28

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.28 добавляет первый каркас полного векторного баланса всей буйковой постановки. Важно: это ещё не итерационный solver равновесной формы. На этом шаге приложение собирает силовые члены в единую ведомость и показывает, какие реакции потребовались бы для замыкания `ΣFx=0` и `ΣFz=0`.

## Что изменилось в v0.28

1. добавлен сервис `MooringVectorBalance`;
2. добавлены силовые члены `MooringVectorBalanceTerm`;
3. добавлен результат `MooringVectorBalanceResult`;
4. в полный баланс включены данные, которые уже есть в расчётном ядре:
   - буй;
   - соединители;
   - приборы;
   - линия;
   - якорь;
   - волновая добавка, если она есть;
5. в диагностике появились строки:
   - `ΣFx учтённых сил постановки`;
   - `ΣFz учтённых сил постановки`;
   - `Требуемая реакция якоря Rx`;
   - `Требуемая реакция якоря Rz`;
6. строка `Полный баланс всей постановки` остаётся `WARNING`, потому что реакции пока не найдены итерационным solver.

## Методология v0.28

Приняты направления:

```text
+X = направление действия течения / горизонтальной нагрузки
+Z = вверх
```

Для каждого элемента из расчётного ядра используется уже существующая строка `ElementCalculationRow`:

```text
Fx = CurrentForceN
Fz = -WeightWaterKg × g
```

Это значит:

```text
положительный WeightWaterKg  → сила вниз
отрицательный WeightWaterKg → плавучесть вверх
```

Волновая добавка берётся из `CalculationResult.WaveForceN` и добавляется как горизонтальная сила.

После этого считаются суммы:

```text
ΣFx учтённых сил постановки
ΣFz учтённых сил постановки
```

и требуемые реакции:

```text
Rx_required = -ΣFx
Rz_required = -ΣFz
```

Эти реакции не подставляются обратно в форму линии и не считаются физической сходимостью. Это только честная векторная ведомость для подготовки следующего solver.

## Что осталось от v0.27.1

Строки внутреннего контроля линии остаются информационными:

```text
ΣFx линии
ΣFz линии
Контроль накопления ΣFx линии
Контроль накопления ΣFz линии
```

Они не являются полной проверкой равновесия всей системы. Это внутренний контроль накопления сил текущей сегментной модели линии.

## Важное ограничение

v0.28 ещё не решает нелинейную равновесную форму:

```text
форма линии ⇄ натяжения ⇄ силы ⇄ реакции ⇄ новое положение узлов
```

Пока полный баланс специально выводится как предупреждение:

```text
Полный баланс всей постановки | каркас собран; реакции не решены | WARNING
```

Это правильно для инженерно-научного приложения: отчёт показывает только то, что уже реально есть в расчётном ядре, и не маскирует незавершённый solver статусом `OK`.

## Что осталось от v0.26

В отчёте остаётся раздел:

```text
## Контроль инженерной модели
```

Он включает геометрию, запасы, состояние буя, силовые информационные строки, векторную ведомость постановки и предупреждение о неполном балансе.

## Что осталось от v0.25.1

Основной 2D-вид использует равный масштаб:

```text
xScale = zScale
```

То есть координаты отображаются без искусственного увеличения по горизонтали.

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Открыть `Отчёт текстом...`.
4. Найти раздел `Контроль инженерной модели`.
5. Проверить, что появились строки:
   - `ΣFx линии`;
   - `ΣFz линии`;
   - `Контроль накопления ΣFx линии`;
   - `Контроль накопления ΣFz линии`;
   - `ΣFx учтённых сил постановки`;
   - `ΣFz учтённых сил постановки`;
   - `Требуемая реакция якоря Rx`;
   - `Требуемая реакция якоря Rz`;
   - `Полный баланс всей постановки`.
6. Проверить, что строки накопления и векторной ведомости имеют статус `INFO`.
7. Проверить, что полный баланс всей постановки пока имеет статус `WARNING`.
