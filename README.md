# BuoyCalc Windows v0.39.1

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.39.1 продолжает подготовку итерационного solver. В v0.39 был добавлен отдельный feedback-цикл, а в v0.39.1 добавлен отдельный сервис сравнения основной формы и финальной кандидатной формы итерационного solver.

Главный принцип остаётся прежним: старый `MooringShapeSolver` не заменяется, 2D/PDF-схемы пока не переключаются на новую форму.

## Что уже есть в v0.39

Итерационный диагностический цикл:

```text
форма → силы по форме → натяжения → дискретные нагрузки → новая форма → сравнение со старой → критерий сходимости
```

Основные типы:

```text
MooringIterativeSolverIteration
MooringIterativeSolverResult
MooringIterativeSolverStore
```

В полном отчёте есть раздел:

```text
## Итерационный solver v0.39 — диагностика
```

## Что добавлено в v0.39.1

1. Добавлен сервис:

```text
Services/MooringIterativeShapeComparison.cs
```

2. Добавлены типы:

```text
MooringIterativeShapeComparisonRow
MooringIterativeShapeComparisonResult
```

3. Сервис сравнивает:

- основную форму `MooringShapeSolver`;
- финальную кандидатную форму `MooringIterativeSolverResult.FinalShape`.

4. Для каждого узла кандидатной формы рассчитываются:

- X/Z основной формы;
- X/Z кандидатной формы;
- ΔX;
- ΔZ;
- полное отклонение узла;
- натяжение основной формы;
- натяжение кандидатной формы;
- ΔT.

5. Итог сравнения содержит:

- основной X-снос;
- кандидатный X-снос;
- ΔX-сноса;
- max Δузла;
- средний Δузла;
- max ΔT;
- общий статус допуска.

6. В `AppInfo` отображаемая версия обновлена:

```text
v0.39.1 - iterative shape comparison
```

## Что не изменилось

1. Основная форма X/Z всё ещё берётся из `MooringShapeSolver`.
2. Новый сравниватель пока является сервисом подготовки к отчётной интеграции.
3. 2D-схема не переключена на кандидатную форму.
4. PDF-схемы не переключены на кандидатную форму.
5. Косметика отчёта не менялась.

## Как проверить

1. Обновить проект локально:

```text
Git → Pull
Build → Clean Solution
Build → Rebuild Solution
```

2. Убедиться, что проект собирается.
3. Проверить CI status:

```text
BuoyCalc Windows Build: success
```

## Следующий шаг

Следующий шаг — подключить `MooringIterativeShapeComparison` к `ReportBuilder`, чтобы в полном отчёте появилась отдельная таблица сравнения основной и кандидатной v0.39-формы. После этого можно будет проверять новый solver на сценариях:

- нормальная поверхностная постановка;
- короткая линия / погружённый буй;
- тяжёлые дискретные приборы;
- большое течение.
