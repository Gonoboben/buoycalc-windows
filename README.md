# BuoyCalc Windows v0.45.2

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.45.2 — правка PDF-отчёта: основная/fallback форма больше не выводится в PDF. Если доступна форма с дискретными элементами, PDF показывает именно её как расчётную форму отчёта. Fallback остаётся внутренней страховкой расчёта, но не объектом PDF-вывода.

## Что изменено

1. Обновлён `Services/PdfReportBuilder.cs`.
2. Убрана PDF-страница сравнения основной и альтернативной формы.
3. На странице схемы постановки PDF теперь рисует форму из `MooringAlternativeShapeStore`, если она доступна.
4. Если форма с дискретными элементами недоступна, PDF пишет об этом текстом и не рисует fallback-форму.
5. Полный текстовый отчёт в PDF фильтруется: блок `Выбор основной формы` и строки `primaryShape...` не выводятся.
6. Обновлён `Services/PdfReportStructureGuide.cs`.
7. Новый заголовок структуры:

```text
## Структура PDF-отчёта v0.45.2
```

8. Версия приложения:

```text
v0.45.2 - PDF discrete shape
```

## Что не изменилось

1. Физика расчёта не изменилась.
2. Solver не изменился.
3. Fallback-форма не удалена из приложения.
4. Gate не удалён.
5. PDF не придумывает координаты: форма берётся из расчётного слоя дискретных нагрузок.

## Как проверить

1. Выполнить расчёт.
2. Экспортировать PDF.
3. На странице схемы должна быть только форма с дискретными элементами.
4. В PDF не должно быть отдельной страницы `Сравнение форм X/Z`.
5. В полном отчёте PDF не должно быть блока `Выбор основной формы`.
6. Проверить CI:

```text
BuoyCalc Windows Build: success
```
