# BuoyCalc Windows v0.24.2

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.24.2 привязывает нижний граничный узел X/Z к проектной глубине. Якорь на 2D-схеме теперь должен находиться на дне, потому что расчётная постановка начинается от якоря.

## Что изменилось в v0.24.2

1. обновлён `MooringNodeAnalyzer`;
2. добавлен расчёт `Build(result, anchorDepthM)`;
3. `ReportBuilder` теперь строит X/Z-узлы с использованием `environment.DepthM`;
4. последний узел `Якорь / нижний конец линии` получает `Z = Depth`;
5. для короткой линии верхний узел остаётся под водой;
6. для длинной линии верхний узел может быть у поверхности, а якорь остаётся на дне;
7. в итогах отчёта добавлена строка `Глубина якорного узла X/Z`.

## Почему это важно

Предыдущая логика брала глубину нижнего узла из последней оценочной глубины сегмента. Это могло поставить якорь выше дна. Теперь нижнее граничное условие задано явно:

```text
якорь = нижняя фиксированная точка
Z якоря = Depth
```

Дальше форма линии строится относительно этого нижнего граничного узла.

## Что осталось от v0.24.1

В 2D-схеме сохраняются разные масштабы:

```text
Z-масштаб = по глубине воды
X-масштаб = по горизонтальному диапазону линии
```

Это нужно, чтобы длинная линия помещалась по ширине, но якорь не зависал в толще воды из-за сжатия вертикального масштаба.

## Как проверить

1. Запустить программу.
2. Поставить глубину `100 м`.
3. Поставить линию `160 м`.
4. Выполнить расчёт.
5. Открыть `Отчёт текстом...`.
6. В разделе `Расчётные узлы линии X/Z` проверить, что последний узел якоря имеет `Z = 100 м`.
7. Открыть `2D-схема`.
8. Проверить, что якорь находится на дне.
