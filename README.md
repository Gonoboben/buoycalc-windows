# BuoyCalc Windows v0.36

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.36 добавляет дискретные X/Z-узлы альтернативной формы. Это следующий слой после v0.35.1: соединители и приборы уже влияют на натяжения и альтернативную форму через координату `s`, а теперь получают собственные координаты `X/Z` на этой альтернативной форме.

Основной solver и основная 2D-визуализация пока не заменяются. Новый слой нужен для проверки отчёта и подготовки будущего режима сравнения форм.

## Что изменилось в v0.36

1. Добавлен сервис `MooringAlternativeDiscreteNodeProjector`.
2. Он берёт:
   - позиционную модель `MooringSequencePositionResult`;
   - альтернативную форму `MooringDiscreteLoadShapeResult`;
   - основную форму `MooringShapeResult`.
3. Для каждого точечного элемента по его координате `s` вычисляются:
   - `X/Z` на альтернативной форме;
   - `X/Z` на основной форме;
   - `ΔX`;
   - `ΔZ`;
   - вес в воде;
   - горизонтальная сила `Fx`;
   - роль узла.
4. Буй и якорь показываются как граничные узлы.
5. Соединители, вертлюги, приборы и другие точечные элементы показываются как внутренние дискретные X/Z-точки.
6. В отчёте появился раздел:

```text
## Дискретные X/Z-узлы альтернативной формы
```

7. В `Итоги` добавлена строка:

```text
Дискретных X/Z-узлов альтернативной формы
```

8. В разделе `Область учёта элементов в текущей модели` появилась колонка:

```text
Дискретный X/Z-узел
```

9. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.36 - alt XZ nodes
```

## Что это даёт

Теперь отчёт может показать не только то, что прибор или соединитель имеет позицию `s`, но и где эта позиция находится в координатах формы:

```text
s -> X/Z основной формы
s -> X/Z альтернативной формы
```

Это важно для будущей визуализации, потому что 2D-схема сможет рисовать приборы и соединители не равномерно по линии, а в реальных расчётных точках.

## Что v0.36 ещё не делает

v0.36 не заменяет основной solver. Дискретные X/Z-узлы пока являются отчётным слоем:

```text
позиция s -> интерполяция по альтернативной форме -> координаты X/Z элемента
```

Они ещё не участвуют как отдельные узлы в итерационном solver и не выводятся отдельным режимом на основной 2D-схеме.

Полный баланс всей постановки остаётся `WARNING`.

## Что осталось от v0.35.1

Контроль альтернативной формы по `angleScale` сохраняется:

```text
|Zякоря - Depth| <= 0.01 м
angleScale <= 2.0
```

Если `angleScale > 2.0`, альтернативная форма получает `WARNING`, даже если геометрически попала в глубину.

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...`.
3. В `Итоги` проверить строку:

```text
Дискретных X/Z-узлов альтернативной формы
```

4. Найти раздел:

```text
## Дискретные X/Z-узлы альтернативной формы
```

5. Проверить, что в таблице есть колонки:
   - `s, м`;
   - `X альт., м`;
   - `Z альт., м`;
   - `X осн., м`;
   - `Z осн., м`;
   - `ΔX, м`;
   - `ΔZ, м`;
   - `Вес в воде, кг`;
   - `Fx, Н`.
6. Проверить, что ADCP, скобы и вертлюг отображаются как внутренние дискретные X/Z-точки.
7. Убедиться, что `Полный баланс всей постановки` пока остаётся `WARNING`.

## Следующий шаг

Рекомендуемый v0.37:

1. добавить отдельный режим 2D-визуализации альтернативной формы;
2. рисовать дискретные X/Z-узлы на этой форме;
3. показывать основную и альтернативную форму как сравнение, не смешивая их физически;
4. после проверки нескольких PDF перейти к итерационному solver:

```text
форма -> силы -> натяжения -> дискретные узлы -> новая форма -> сходимость
```
