# BuoyCalc Windows v0.31

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.31 добавляет отдельную инженерную ведомость shape-based сил линии. Теперь отчёт не только показывает форму X/Z и проекции сегментов, но и сравнивает старую силу сегмента с новой оценкой по нормальной составляющей скорости к фактической ориентации сегмента.

Это подготовка к будущему итерационному solver:

```text
форма X/Z -> проекции сегментов -> нормальная скорость -> shape-based силы -> будущие натяжения -> новая форма
```

## Что изменилось в v0.31

1. Добавлен сервис `MooringShapeForceAnalyzer`.
2. Для каждого сегмента линии считаются:
   - локальная скорость `U`;
   - нормальная к сегменту скорость `Uнорм`;
   - старая сила сегмента из `SegmentCalculationRow`;
   - shape-based сила сегмента;
   - разница `ΔF`;
   - отношение `Ratio`;
   - статус строки.
3. В отчёт добавлен раздел:

```text
## Shape-based силы линии по ориентации сегментов
```

4. В итогах отчёта появились строки:
   - `Shape-based сила линии`;
   - `Отличие shape-based силы линии`;
   - `Статус shape-based сил`.

## Методика v0.31

Для каждого сегмента берётся фактическая ориентация из формы X/Z:

```text
t = (dX / L, dZ / L)
```

Скорость течения представляется в плоскости X/Z:

```text
U = (Uгор, W)
```

Затем считается нормальная составляющая скорости:

```text
Uнорм = sqrt(|U|^2 - dot(U, t)^2)
```

И shape-based сила:

```text
Fshape = 0.5 * rho * Cd * A * Uнорм^2
```

Сейчас эта сила только сравнивается со старой оценкой и ещё не подставляется обратно в основной расчёт.

## Что осталось от v0.30

В отчёте остаётся раздел:

```text
## Проекции формы X/Z по сегментам
```

Он проверяет:

```text
ΣdX = Xякоря - Xбуя
ΣdZ = Zякоря - Zбуя
ΣLпо координатам ≈ ΣLсегментов
```

## Что осталось от v0.29

`MooringShapeSolver` имеет геометрический критерий сходимости:

```text
|Zякоря - Depth| <= 0.01 м
```

В диагностике остаются строки:

```text
Итерации solver формы
Невязка сходимости формы
Статус сходимости формы
```

## Важное ограничение

v0.31 ещё не замыкает полный нелинейный solver:

```text
форма линии <-> локальные силы <-> натяжения <-> углы <-> новая форма <-> реакции
```

Поэтому `Полный баланс всей постановки` должен оставаться `WARNING`.

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...`.
3. В `Итоги` проверить строки:
   - `Shape-based сила линии`;
   - `Отличие shape-based силы линии`;
   - `Статус shape-based сил`.
4. Найти раздел:

```text
## Shape-based силы линии по ориентации сегментов
```

5. Проверить, что таблица содержит `Uнорм`, `F старая`, `F shape`, `ΔF`, `Ratio`.
6. Убедиться, что `Полный баланс всей постановки` пока остаётся `WARNING`.

## Следующий шаг

Рекомендуемый v0.32:

1. использовать shape-based силы как вход для нового расчёта натяжений;
2. построить альтернативную таблицу натяжений от shape-based сил;
3. сравнить старые и новые натяжения;
4. после этого начать итерацию `форма -> силы -> натяжения -> новая форма`.
