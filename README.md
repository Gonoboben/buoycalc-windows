# BuoyCalc Windows v0.46.3

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.46.3 — первый пакет приведения проекта к рабочему состоянию после контрольной отметки. Пользовательский PDF остаётся коротким, а полный технический отчёт остаётся отдельным окном. В этом шаге уточнены термины отчёта и русифицирован блок автопроверок.

Расчётная физика, solver, 2D-координаты и форма с дискретными элементами не изменены.

## Что изменено

1. Окно текстового отчёта переименовано в:

```text
Полный отчёт
```

2. Обновлён файл:

```text
Views/ReportTextWindow.axaml
```

3. Блок автопроверок в полном отчёте русифицирован:

```text
Services/MooringAutocheckSuite.cs
```

4. Старый заголовок:

```text
Scenario and element database autochecks v0.43
```

заменён на:

```text
Автопроверки сценариев и базы элементов v0.46.3
```

5. Статусы таблицы теперь выводятся по-русски:

```text
ОК
Информация
Предупреждение
Ошибка
```

6. Итог автопроверок теперь тоже на русском:

```text
ОК: базовые автопроверки пройдены
ПРЕДУПРЕЖДЕНИЕ: критичных ошибок нет, предупреждений: ...
ОШИБКА: критичных проверок не пройдено: ...
```

7. Версия приложения:

```text
v0.46.3 - полный отчёт и русские автопроверки
```

## Что пока не изменено

1. Основная кнопка в верхней панели главного окна ещё может называться `Отчёт текстом...`; большая XAML-правка главного окна была отложена, чтобы не рисковать сборкой.
2. Предупреждения на 2D/схеме пока не переведены в новую пользовательскую warning policy.
3. 3D-окно пока не добавлялось.
4. Физика solver пока не усиливалась.

## Что не изменилось

1. Физика расчёта не изменилась.
2. Solver не изменился.
3. Gate не удалён.
4. Fallback-форма не удалена из приложения.
5. PDF-схема берёт координаты из расчётного слоя дискретных нагрузок.
6. Release build workflow остаётся тем же.

## Как проверить

1. Выполнить расчёт.
2. Открыть окно `Полный отчёт`.
3. Найти раздел:

```text
Автопроверки сценариев и базы элементов v0.46.3
```

4. Проверить, что в этом разделе больше нет английских строк `Scenario`, `Expected`, `Result`, `Warning`, `Fail`.
5. Проверить CI:

```text
BuoyCalc Windows Build: success
```
