# BuoyCalc Windows v0.13.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.13.1 переносит визуальную схему постановки в отдельное окно подтверждения. Теперь при нажатии `Проверить схему и рассчитать` программа сначала показывает схему, а расчёт выполняется только после подтверждения пользователя.

## Что изменилось в v0.13.1

1. визуальная схема убрана из центральной колонки главного окна;
2. добавлено отдельное окно `Проверка схемы постановки`;
3. кнопка расчёта теперь открывает окно проверки схемы;
4. в окне проверки есть две кнопки:
   - `Вернуться`;
   - `Всё верно, рассчитать`;
5. если пользователь нажимает `Вернуться`, расчёт не выполняется;
6. если пользователь нажимает `Всё верно, рассчитать`, запускается расчёт;
7. схема показывает только активные элементы с галочкой `Учитывать в расчёте`;
8. решение сделано без внешних пакетов.

## Логика перед расчётом

```text
Нажать "Проверить схему и рассчитать"
↓
Открывается окно схемы
↓
Пользователь проверяет порядок элементов
↓
Если всё верно — подтверждает
↓
Только после этого выполняется расчёт
```

## Как проверить

1. Запустить программу.
2. Нажать кнопку `Проверить схему и рассчитать`.
3. Проверить, что открылось отдельное окно `Проверка схемы постановки`.
4. Нажать `Вернуться`.
5. Проверить, что расчёт не выполнился.
6. Снова нажать `Проверить схему и рассчитать`.
7. Нажать `Всё верно, рассчитать`.
8. Проверить, что расчёт выполнился и появились результат, таблица и отчёт.
9. Отключить элемент галочкой `Учитывать в расчёте`.
10. Повторить проверку и убедиться, что отключённый элемент не отображается в схеме.
