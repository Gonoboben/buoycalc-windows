# BuoyCalc Windows v0.32

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.32 добавляет альтернативную ведомость натяжений, рассчитанную от shape-based сил линии. Теперь отчёт показывает не только старые натяжения и shape-based силы, но и то, как эти силы меняют накопленное натяжение снизу вверх.

Это следующий промежуточный слой перед настоящей итерацией формы:

```text
форма X/Z -> проекции -> shape-based силы -> shape-based натяжения -> будущая новая форма
```

## Что изменилось в v0.32

1. Добавлен сервис `MooringShapeTensionAnalyzer`.
2. Он повторяет принцип старого `SegmentTensionAnalyzer`: расчёт идёт снизу вверх, от якоря к бую.
3. Вместо старой силы сегмента `SegmentCalculationRow.CurrentForceN` используется `ShapeForceN` из `MooringShapeForceAnalyzer`.
4. Для каждого сегмента сравниваются:
   - старая сила;
   - shape-based сила;
   - старое натяжение;
   - shape-based натяжение;
   - разница натяжений;
   - старый угол;
   - shape-based угол;
   - разница углов.
5. В отчёт добавлен раздел:

```text
## Shape-based натяжения линии
```

6. В итогах отчёта появились строки:
   - `Top T старая / shape`;
   - `Макс. отличие shape-based натяжения`;
   - `Статус shape-based натяжений`.
7. Отображаемая версия окон обновлена:

```text
v0.32 - shape tension report
```

## Что v0.32 ещё не делает

v0.32 пока не перестраивает форму по shape-based натяжениям. Это сделано намеренно: сначала нужно увидеть, насколько новая силовая модель меняет натяжения, и только потом подключать её в итерационный solver.

Полная схема ещё не замкнута:

```text
форма линии <-> локальные силы <-> натяжения <-> углы <-> новая форма <-> реакции
```

Поэтому `Полный баланс всей постановки` должен оставаться `WARNING`.

## Что осталось от v0.31

В отчёте остаётся раздел:

```text
## Shape-based силы линии по ориентации сегментов
```

Он считает нормальную скорость к сегменту и shape-based силу:

```text
Fshape = 0.5 * rho * Cd * A * Uнорм^2
```

## Что осталось от v0.30

В отчёте остаётся раздел:

```text
## Проекции формы X/Z по сегментам
```

Он проверяет геометрию формы:

```text
ΣdX = Xякоря - Xбуя
ΣdZ = Zякоря - Zбуя
ΣLпо координатам ≈ ΣLсегментов
```

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...`.
3. В `Итоги` проверить строки:
   - `Top T старая / shape`;
   - `Макс. отличие shape-based натяжения`;
   - `Статус shape-based натяжений`.
4. Найти раздел:

```text
## Shape-based натяжения линии
```

5. Проверить, что таблица содержит:
   - `F старая`;
   - `F shape`;
   - `T старая`;
   - `T shape`;
   - `ΔT`;
   - `Угол старый`;
   - `Угол shape`;
   - `Δугла`.
6. Убедиться, что `Полный баланс всей постановки` пока остаётся `WARNING`.

## Следующий шаг

Рекомендуемый v0.33:

1. построить экспериментальный solver-цикл:

```text
форма -> shape-based силы -> shape-based натяжения -> новые углы -> новая форма
```

2. добавить критерий сходимости по изменению формы и натяжений;
3. оставить старый solver как fallback;
4. только после этого постепенно переводить `Полный баланс всей постановки` из предупреждения в реальную проверку.
