# BuoyCalc Windows v0.28.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.28.1 добавляет в текстовый отчёт подробную таблицу векторной ведомости сил постановки. Теперь отчёт показывает не только итоговые `ΣFx` / `ΣFz` и требуемые реакции, но и вклад каждого силового члена: буй, линия, соединители, приборы, якорь и волновая добавка.

## Что изменилось в v0.28.1

1. в `ReportBuilder` добавлен раздел `Векторная ведомость сил постановки`;
2. таблица строится из `MooringVectorBalance.Terms`;
3. в таблице выводятся:
   - номер строки;
   - тип элемента;
   - имя элемента;
   - `Fx, Н`;
   - `Fz, Н`;
   - методическое примечание;
4. внизу таблицы выводятся:
   - сумма учтённых сил `ΣFx`, `ΣFz`;
   - требуемая реакция для замыкания `ΣF=0`;
   - горизонтальная удерживающая способность якоря;
   - запас горизонтального удержания по `Rx`;
   - статус баланса;
5. раздел ограничений теперь явно пишет, что подробная ведомость есть, но итерационный solver равновесной формы ещё не включён.

## Что осталось от v0.28

v0.28 добавил первый каркас полного векторного баланса всей буйковой постановки. Это ещё не итерационный solver равновесной формы. На этом шаге приложение собирает силовые члены в единую ведомость и показывает, какие реакции потребовались бы для замыкания `ΣFx=0` и `ΣFz=0`.

В полный баланс включены данные, которые уже есть в расчётном ядре:

```text
буй
соединители
приборы
линия
якорь
волновая добавка, если она есть
```

## Методология v0.28 / v0.28.1

Приняты направления:

```text
+X = направление действия течения / горизонтальной нагрузки
+Z = вверх
```

Для каждого элемента из расчётного ядра используется уже существующая строка `ElementCalculationRow`:

```text
Fx = CurrentForceN
Fz = -WeightWaterKg × g
```

Это значит:

```text
положительный WeightWaterKg  → сила вниз
отрицательный WeightWaterKg → плавучесть вверх
```

Волновая добавка берётся из `CalculationResult.WaveForceN` и добавляется как горизонтальная сила.

После этого считаются суммы:

```text
ΣFx учтённых сил постановки
ΣFz учтённых сил постановки
```

и требуемые реакции:

```text
Rx_required = -ΣFx
Rz_required = -ΣFz
```

Эти реакции не подставляются обратно в форму линии и не считаются физической сходимостью. Это только честная векторная ведомость для подготовки следующего solver.

## Что осталось от v0.27.1

Строки внутреннего контроля линии остаются информационными:

```text
ΣFx линии
ΣFz линии
Контроль накопления ΣFx линии
Контроль накопления ΣFz линии
```

Они не являются полной проверкой равновесия всей системы. Это внутренний контроль накопления сил текущей сегментной модели линии.

## Важное ограничение

v0.28.1 ещё не решает нелинейную равновесную форму:

```text
форма линии ⇄ натяжения ⇄ силы ⇄ реакции ⇄ новое положение узлов
```

Пока полный баланс специально выводится как предупреждение:

```text
Полный баланс всей постановки | каркас собран; реакции не решены | WARNING
```

Это правильно для инженерно-научного приложения: отчёт показывает только то, что уже реально есть в расчётном ядре, и не маскирует незавершённый solver статусом `OK`.

## Что осталось от v0.26

В отчёте остаётся раздел:

```text
## Контроль инженерной модели
```

После него теперь идёт раздел:

```text
## Векторная ведомость сил постановки
```

Он показывает подробную расшифровку `ΣFx` и `ΣFz` по силовым членам.

## Что осталось от v0.25.1

Основной 2D-вид использует равный масштаб:

```text
xScale = zScale
```

То есть координаты отображаются без искусственного увеличения по горизонтали.

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Открыть `Отчёт текстом...`.
4. Найти раздел `Контроль инженерной модели`.
5. Проверить строки:
   - `ΣFx учтённых сил постановки`;
   - `ΣFz учтённых сил постановки`;
   - `Требуемая реакция якоря Rx`;
   - `Требуемая реакция якоря Rz`;
   - `Полный баланс всей постановки`.
6. Ниже найти раздел `Векторная ведомость сил постановки`.
7. Проверить, что в таблице есть строки по силовым членам и итоговые строки:
   - `Сумма учтённых сил`;
   - `Требуемая реакция для ΣF=0`.
8. Проверить, что полный баланс всей постановки пока имеет статус `WARNING`.
