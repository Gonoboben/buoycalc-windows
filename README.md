# BuoyCalc Windows v0.22.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.22.1 уточняет таблицу расчётных узлов `X/Z`: буй и якорь теперь явно показаны как граничные узлы линии.

## Что изменилось в v0.22.1

1. верхний узел теперь подписан как `Буй / верхний конец линии`;
2. у верхнего узла показывается натяжение и угол первого сегмента;
3. последний узел теперь подписан как `Якорь / нижний конец линии`;
4. у последнего узла показывается натяжение последнего сегмента;
5. в отчёт добавлено пояснение, что буй и якорь — граничные узлы, а не сегменты троса;
6. ограничение отчёта обновлено до `v0.22.1`.

## Что было добавлено в v0.22

1. добавлен сервис `MooringNodeAnalyzer`;
2. добавлена модель `MooringNodeRow`;
3. расчёт строит узлы сверху вниз:
   - верхний узел линии;
   - узел после каждого сегмента;
   - накопленная длина вдоль линии `s`;
   - предварительная координата `X`;
   - предварительная глубина `Z`;
   - длина сегмента;
   - угол сегмента;
   - натяжение сегмента;
4. в разделе `Итоги` появился предварительный горизонтальный снос по узлам `X/Z`;
5. в отчёт добавлен раздел `Расчётные узлы линии X/Z`.

## Что важно

Это ещё не финальный `moordyn` и не полная равновесная форма. Узлы строятся по текущим оценочным углам сегментов:

```text
ΔX = L × sin(угол)
ΔZ = L × cos(угол)
```

Затем глубина нормируется по оценочной глубине сегментов. Это переходная модель, нужная для следующего этапа — итерационного равновесного расчёта и замены 2D-схемы.

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Нажать `Отчёт текстом...`.
4. Найти раздел `Расчётные узлы линии X/Z`.
5. Проверить, что первый узел подписан как `Буй / верхний конец линии`.
6. Проверить, что последний узел подписан как `Якорь / нижний конец линии`.
7. Проверить, что у буя показано натяжение первого сегмента, а у якоря — последнего сегмента.
