# BuoyCalc Windows v0.43

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.43 выполняет этап плана: улучшение базы элементов. Сделано безопасно: сами элементы, UI и расчётная физика не изменены. Вместо этого добавлены проверки качества базы элементов в существующий слой автопроверок.

Старый `MooringShapeSolver` не удалён и остаётся fallback.

## Что уже есть в v0.42

1. Добавлен сервис автопроверок:

```text
Services/MooringAutocheckSuite.cs
```

2. В полный отчёт выводится таблица автопроверок.

3. Базовые сценарии:

```text
input-geometry
shape-fallback
shape-candidate
iterative-solver
deployment-mode
loads
primary-selection
```

## Что добавлено в v0.43

1. Существующий сервис `MooringAutocheckSuite` расширен сценарием:

```text
element-database
```

2. Проверки базы элементов ищут:

```text
пустые названия элементов
пустые типы элементов
отсутствующие preset names
неположительные количества
отрицательные длины
отсутствие распределённых line-строк
площадь без Cd
Cd без площади при наличии силы течения
WLL больше MBL
отрицательные MBL/WLL
неконечные значения reserve
```

3. Проверки не изменяют элементы и не блокируют расчёт. Они только показывают качество текущей базы в отчёте.

4. Заголовок таблицы автопроверок обновлён:

```text
## Scenario and element database autochecks v0.43
```

5. В `AppInfo` отображаемая версия обновлена:

```text
v0.43 - element database autochecks
```

## Что не изменилось

1. Физика расчёта не изменилась.
2. UI базы элементов не изменён.
3. Сами пресеты элементов не переписаны.
4. Старый `MooringShapeSolver` не удалён.
5. 2D/PDF продолжают брать основную форму из `MooringShapeStore`.
6. UX редактора последовательности остаётся задачей v0.44.

## Как проверить

1. Обновить проект локально:

```text
Git -> Pull
Build -> Clean Solution
Build -> Rebuild Solution
```

2. Выполнить обычный расчёт.
3. В полном отчёте найти:

```text
Scenario and element database autochecks v0.43
```

4. Проверить строки со сценарием:

```text
element-database
```

5. Проверить CI status:

```text
BuoyCalc Windows Build: success
```

## Следующий безопасный шаг

Если v0.43 проходит CI, следующий этап по плану — v0.44: UX редактора последовательности.
