# BuoyCalc Windows v0.39

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.39 начинает отдельный каркас итерационного solver. Старый `MooringShapeSolver` остаётся основным источником формы для 2D и PDF, а новый слой работает как диагностический feedback-цикл.

Цель версии — не заменить физику сразу, а замкнуть уже существующие расчётные блоки:

```text
форма → силы по форме → натяжения → дискретные нагрузки → новая форма → сравнение со старой → критерий сходимости
```

## Что изменилось в v0.39

1. Добавлен отдельный сервис:

```text
Services/MooringIterativeSolver.cs
```

2. Добавлены отдельные результаты:

```text
MooringIterativeSolverIteration
MooringIterativeSolverResult
MooringIterativeSolverStore
```

3. Новый слой запускается из `ReportBuilder` после расчёта основной формы, shape-based сил, shape-based натяжений, позиционной модели и альтернативной формы.

4. Основной solver не удалён и не заменён:

```text
MooringShapeSolver → остаётся основной формой для 2D/PDF
MooringIterativeSolver → диагностический слой v0.39
```

5. В полный отчёт добавлен раздел:

```text
## Итерационный solver v0.39 — диагностика
```

6. Диагностическая таблица показывает по итерациям:

- входной X-снос;
- shape-based силу линии;
- верхнее shape-based натяжение;
- верхнее натяжение с дискретными нагрузками;
- новый X-снос;
- ΔX;
- max Δузла;
- невязку Z;
- статус геометрии;
- статус итерации.

7. В `AppInfo` отображаемая версия обновлена:

```text
v0.39 - iterative solver skeleton
```

## Что не изменилось

1. Старая основная форма X/Z не заменена.
2. 2D-схема продолжает показывать основную и альтернативную формы, как раньше.
3. PDF-схемы продолжают показывать старую основную и альтернативную форму.
4. Новый solver пока используется только как диагностический слой в отчёте.
5. Косметика отчёта пока не менялась.

## Как проверить

1. Обновить проект локально:

```text
Git → Pull
Build → Clean Solution
Build → Rebuild Solution
```

2. Запустить расчёт стандартного проекта.
3. В полном отчёте найти раздел:

```text
Итерационный solver v0.39 — диагностика
```

4. Экспортировать PDF и проверить, что в полном текстовом отчёте PDF этот раздел тоже присутствует.
5. Убедиться, что первая PDF-схема и страница сравнения форм не изменили свою логику.
6. Проверить CI status:

```text
BuoyCalc Windows Build: success
```

## Следующий шаг

После проверки v0.39 можно переходить к аккуратному использованию результата итерационного solver как кандидата на новую расчётную форму, но только после сравнения на нескольких сценариях: нормальная поверхностная постановка, короткая линия / погружённый буй, постановка с тяжёлыми дискретными приборами и постановка с большим течением.
