# BuoyCalc Windows v0.26

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.26 добавляет инженерную диагностику расчётной модели. Теперь отчёт показывает не только результат, но и контроль качества: невязки, статусы, запасы и предупреждение о том, что текущая форма пока предварительная.

## Что изменилось в v0.26

1. добавлен сервис `EngineeringDiagnostics`;
2. добавлены строки диагностики `EngineeringDiagnosticRow`;
3. добавлен общий результат диагностики `EngineeringDiagnosticsResult`;
4. добавлены статусы:
   - `INFO`;
   - `OK`;
   - `WARNING`;
   - `ERROR`;
5. `ReportBuilder` теперь строит диагностику после `MooringShapeSolver`;
6. в отчёте появился раздел `Контроль инженерной модели`;
7. в итогах отчёта появилась строка `Диагностика`;
8. в начале отчёта появился общий статус инженерной диагностики.

## Что проверяет v0.26

```text
1. Якорь на проектной глубине
2. Длина расчётной линии
3. Состояние буя
4. Геометрия: линия / глубина
5. Положение буя выше якоря
6. Максимальное натяжение
7. Запас слабого звена
8. Запас якоря
9. Статус сходимости формы
```

В отчёте это выводится таблицей:

```text
| Проверка | Значение | Допуск / критерий | Статус | Примечание |
```

## Почему это важно

Инженерная программа должна показывать не только форму и итоговый вердикт, но и качество модели:

```text
расчёт
↓
форма X/Z
↓
невязки и проверки
↓
отчёт / 2D / PDF
```

Если модель предварительная, отчёт должен явно говорить об этом. Сейчас поле сходимости формы остаётся:

```text
Converged = false
```

Это правильно: координаты уже вынесены в инженерный объект, но полный итерационный solver равновесной формы ещё не включён.

## Что осталось от v0.25.1

Основной 2D-вид использует равный масштаб:

```text
xScale = zScale
```

То есть координаты отображаются без искусственного увеличения по горизонтали.

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Открыть `Отчёт текстом...`.
4. Найти раздел `Контроль инженерной модели`.
5. Проверить, что есть строки:
   - `Якорь на проектной глубине`;
   - `Длина расчётной линии`;
   - `Состояние буя`;
   - `Запас слабого звена`;
   - `Запас якоря`;
   - `Статус сходимости формы`.
6. Проверить, что статусы выводятся как `OK`, `WARNING`, `ERROR` или `INFO`.
