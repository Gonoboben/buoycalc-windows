# BuoyCalc Windows v0.42

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.42 выполняет этап плана: добавлены тестовые сценарии и автопроверки. Сделано безопасно: это встроенный diagnostic/autocheck слой, который не меняет solver, силы, натяжения, 2D или PDF-геометрию, а проверяет согласованность текущего расчёта и выводит результат в полный отчёт.

Старый `MooringShapeSolver` не удалён и остаётся fallback.

## Что уже есть в v0.41

1. Добавлен классификатор режима постановки:

```text
Services/MooringDeploymentModeClassifier.cs
```

2. Поддерживаемые режимы:

```text
surface
submerged
short
excess line
overloaded
unknown
```

3. В полный отчёт выводится таблица:

```text

```

## Что добавлено в v0.42

1. Добавлен сервис автопроверок:

```text
Services/MooringAutocheckSuite.cs
```

2. Добавлены типы:

```text
MooringAutocheckSeverity
MooringAutocheckRow
MooringAutocheckResult
```

3. Автопроверки выполняются по текущему расчёту и проверяют:

```text
input-geometry
shape-fallback
shape-candidate
iterative-solver
deployment-mode
loads
primary-selection
```

4. В полный отчёт добавлена таблица:

```text
## Автопроверки сценариев v0.42
```

5. Таблица показывает:

```text
№
Сценарий
Проверка
Ожидается
Фактически
Статус
Примечание
```

6. Используются уровни:

```text
Pass
Info
Warning
Fail
```

7. Автопроверки не останавливают расчёт и не меняют физику. Сейчас это встроенный diagnostic слой, который готовит основу для будущих регрессионных тестов.

8. В `AppInfo` отображаемая версия обновлена:

```text
v0.42 - scenario autochecks
```

## Что не изменилось

1. Классификатор режима не меняет физику расчёта.
2. Автопроверки не меняют физику расчёта.
3. Старый `MooringShapeSolver` не удалён.
4. Если кандидатная форма не прошла gate, приложение остаётся на старой форме.
5. 2D/PDF продолжают брать основную форму из `MooringShapeStore`.
6. Улучшение базы элементов остаётся задачей v0.43.

## Как проверить

1. Обновить проект локально:

```text
Git -> Pull
Build -> Clean Solution
Build -> Rebuild Solution
```

2. Выполнить обычный расчёт.
3. В полном отчёте найти:

```text
Автопроверки сценариев v0.42
```

4. Проверить, что в таблице есть сценарии:

```text
input-geometry
shape-fallback
shape-candidate
iterative-solver
deployment-mode
loads
primary-selection
```

5. Проверить CI status:

```text
BuoyCalc Windows Build: success
```

## Следующий безопасный шаг
