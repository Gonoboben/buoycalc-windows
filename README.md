# BuoyCalc Windows v0.27

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.27 добавляет силовую диагностику текущей сегментной модели линии. Теперь отчёт показывает не только геометрические невязки, но и внутренний контроль сумм сил `ΣFx` / `ΣFz` по линии.

## Что изменилось в v0.27

1. расширен `EngineeringDiagnostics`;
2. добавлен результат `EngineeringForceResiduals`;
3. в диагностику добавлены строки:
   - `ΣFx линии`;
   - `ΣFz линии`;
   - `Невязка ΣFx сегментной суммы`;
   - `Невязка ΣFz сегментной суммы`;
   - `Полный баланс всей постановки`;
4. отчёт теперь явно различает:
   - внутренний баланс сегментной модели линии;
   - полный баланс всей буйковой постановки;
5. полный баланс всей постановки пока помечается как `WARNING`, потому что он требует итерационного solver.

## Что именно считается сейчас

Текущая силовая диагностика проверяет внутреннюю согласованность линии:

```text
ΣFx линии = сумма горизонтальных сил сопротивления по сегментам
ΣFz линии = сумма вертикальных весовых сил по сегментам
Ttop_x ≈ ΣFx линии
Ttop_z ≈ ΣFz линии
```

Для этого добавлены относительные невязки:

```text
ResidualFx = |ΣFx - Ttop_x|
RelativeResidualFx = ResidualFx / max(1, |ΣFx|)

ResidualFz = |ΣFz - Ttop_z|
RelativeResidualFz = ResidualFz / max(1, |ΣFz|)
```

## Важное ограничение

Это ещё не полный баланс всей системы:

```text
буй + соединители + приборы + линия + якорь
```

Пока полный баланс специально выводится как предупреждение:

```text
Полный баланс всей постановки | ещё не решается | WARNING
```

Это честно: текущая модель уже контролирует сегментную сумму линии, но общий итерационный solver равновесия ещё не включён.

## Что осталось от v0.26

В отчёте остаётся раздел:

```text
## Контроль инженерной модели
```

Он теперь включает геометрию, запасы, состояние буя и силовые невязки.

## Что осталось от v0.25.1

Основной 2D-вид использует равный масштаб:

```text
xScale = zScale
```

То есть координаты отображаются без искусственного увеличения по горизонтали.

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Открыть `Отчёт текстом...`.
4. Найти раздел `Контроль инженерной модели`.
5. Проверить, что появились строки:
   - `ΣFx линии`;
   - `ΣFz линии`;
   - `Невязка ΣFx сегментной суммы`;
   - `Невязка ΣFz сегментной суммы`;
   - `Полный баланс всей постановки`.
6. Проверить, что полный баланс всей постановки пока имеет статус `WARNING`.
