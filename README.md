# BuoyCalc Windows v0.33

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.33 добавляет позиционную модель последовательности по координате `s` вдоль линии. Теперь соединители, приборы, буй, якорь и участки линии имеют явное место в постановке: линейные элементы занимают интервал `s0–s1`, а дискретные элементы имеют точечную координату `s`.

Это не меняет solver формы напрямую, но закрывает важный методический пробел: теперь у каждого дискретного элемента есть позиция, куда на следующем шаге можно добавить локальный скачок веса и горизонтальной силы.

## Что изменилось в v0.33

1. Добавлен сервис `MooringSequencePositioner`.
2. Для каждого элемента из `ElementRows` строится строка позиционной модели:
   - номер;
   - тип;
   - элемент;
   - пресет;
   - `s0, м`;
   - `s1, м`;
   - `s, м`;
   - длина;
   - вес в воде;
   - горизонтальная сила;
   - роль в solver;
   - следующий шаг.
3. В отчёте появился раздел:

```text
## Позиционная модель последовательности по s
```

4. В `Итоги` добавлена сводка:

```text
Дискретных элементов с координатой s
```

5. В разделе `Область учёта элементов в текущей модели` появилась колонка:

```text
Позиция s
```

6. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.33 - sequence positions
```

## Как работает позиционная модель

Координата `s` отсчитывается вдоль линии от верхнего конца к якорю.

```text
Буй                 -> s = 0
Соединитель над линией -> s = 0
Линия 100 м          -> s0 = 0, s1 = 100
Прибор после линии   -> s = 100
Следующая линия 10 м -> s0 = 100, s1 = 110
Цепь 10 м            -> s0 = 110, s1 = 120
Якорь                -> s = 120
```

Для линий `s` в таблице показывается как середина интервала, а `s0/s1` показывают начало и конец участка.

## Что v0.33 ещё не делает

v0.33 ещё не подставляет дискретные нагрузки обратно в solver формы. Сейчас соединители и приборы:

```text
имеют координату s
учтены в таблице элементов
учтены в векторной ведомости
но ещё не дают локальный скачок натяжения и угла в форме X/Z
```

Полный баланс всей постановки поэтому остаётся `WARNING`.

## Что осталось от v0.32.1

В отчёте остаются методические уточнения:

```text
Полная плавучесть буя
Чистая плавучесть
Старая оценка сноса
Снос формы X/Z
Расчётная нагрузка для проверки слабого звена
```

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...`.
3. Найти раздел:

```text
## Позиционная модель последовательности по s
```

4. Проверить, что:
   - у буя `s = 0`;
   - у верхнего соединителя `s = 0`;
   - у линии есть `s0` и `s1`;
   - у приборов и соединителей после линии `s` равен концу предыдущей линии;
   - у якоря `s` равен полной длине линии.
5. Найти раздел `Область учёта элементов в текущей модели` и проверить колонку `Позиция s`.
6. Убедиться, что `Полный баланс всей постановки` пока остаётся `WARNING`.
