# BuoyCalc Windows v0.34

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.34 использует позиционную модель `s` из v0.33 для расчёта альтернативной ведомости натяжений с локальными нагрузками от соединителей и приборов.

Это первый шаг, где дискретные элементы уже не только имеют координату `s`, но и попадают в расчёт натяжений как добавки веса в воде и горизонтальной силы ниже рассматриваемого сечения линии.

## Что изменилось в v0.34

1. Добавлен сервис `MooringDiscreteLoadTensionAnalyzer`.
2. Он берёт:
   - сегменты линии из `SegmentRows`;
   - старые натяжения из `SegmentTensionAnalyzer`;
   - позиции элементов из `MooringSequencePositioner`.
3. Из позиционной модели выбираются дискретные элементы:

```text
соединители
приборы
другие точечные элементы между буем и якорем
```

Буй и якорь остаются граничными условиями и не добавляются как внутренние дискретные нагрузки.

4. Для каждого сечения линии расчёт идёт снизу вверх. В сечении учитываются:
   - распределённые силы и вес сегментов ниже сечения;
   - вес в воде дискретных элементов ниже или на этом `s`;
   - горизонтальная сила дискретных элементов ниже или на этом `s`.
5. В отчёте появился раздел:

```text
## Натяжения линии с дискретными нагрузками по s
```

6. В итогах отчёта появились строки:
   - `Top T старая / с дискретными нагрузками`;
   - `Макс. отличие натяжения от дискретных нагрузок`;
   - `Статус натяжений с дискретными нагрузками`.
7. В разделе области учёта элементов появилась колонка:

```text
Дискретные натяжения
```

8. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.34 - tension report
```

## Что v0.34 ещё не делает

v0.34 пока не перестраивает форму X/Z по этим новым натяжениям. То есть новая ведомость показывает, как дискретные нагрузки влияют на натяжения, но координаты формы ещё остаются результатом предыдущего solver.

Полная схема ещё не замкнута:

```text
форма X/Z -> s-позиции -> локальные дискретные нагрузки -> натяжения -> новые углы -> новая форма
```

Поэтому `Полный баланс всей постановки` остаётся `WARNING`.

## Что осталось от v0.33

В отчёте остаётся раздел:

```text
## Позиционная модель последовательности по s
```

Он показывает, где расположен каждый элемент вдоль линии.

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...`.
3. В `Итоги` проверить строки:
   - `Top T старая / с дискретными нагрузками`;
   - `Макс. отличие натяжения от дискретных нагрузок`;
   - `Статус натяжений с дискретными нагрузками`.
4. Найти раздел:

```text
## Натяжения линии с дискретными нагрузками по s
```

5. Проверить подраздел `Дискретные нагрузки по s`.
6. Проверить таблицу накопленных натяжений: в ней должны быть колонки:
   - `Вес дискр. ниже`;
   - `Fx дискр. ниже`;
   - `T дискр.`;
   - `Угол дискр.`.
7. Убедиться, что `Полный баланс всей постановки` пока остаётся `WARNING`.

## Следующий шаг

Рекомендуемый v0.35:

1. использовать натяжения с дискретными нагрузками для расчёта новых углов;
2. построить альтернативную форму X/Z с учётом дискретных нагрузок;
3. сравнить старую форму и новую форму;
4. после этого переходить к итерации:

```text
форма -> s-позиции -> дискретные нагрузки -> натяжения -> углы -> новая форма
```
