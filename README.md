# BuoyCalc Windows v0.24.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.24.1 исправляет масштабирование 2D-схемы при очень длинной линии. Теперь вертикальный масштаб `Z` привязан к глубине воды, а горизонтальный масштаб `X` считается отдельно.

## Что изменилось в v0.24.1

1. обновлён `Mooring2DCanvas`;
2. масштаб по `Z` больше не зависит от горизонтального размаха линии;
3. масштаб по `X` считается отдельно, чтобы длинная/сильно снесённая линия помещалась по ширине;
4. якорь больше не должен визуально зависать в толще воды из-за сжатия общего масштаба;
5. на схеме добавлена подпись `X/Z масштабы разные`.

## Что было исправлено в v0.24

v0.24 исправила важную ошибку визуализации и таблицы узлов: верхний узел линии больше не считается всегда находящимся на поверхности. Если линия короче глубины, верхний узел смещается ниже поверхности, и 2D-схема показывает буй под водой.

## Почему это важно

Раньше 2D-схема использовала один общий масштаб для `X` и `Z`:

```text
scale = min(scaleZ, scaleX)
```

Если линия была очень длинной, горизонтальный диапазон мог уменьшить общий масштаб. Из-за этого вся вертикаль сжималась, и якорь визуально оказывался выше дна. Теперь:

```text
Z-масштаб = по глубине воды
X-масштаб = по горизонтальному диапазону линии
```

Это делает схему инженерно читаемой: дно остаётся дном, а горизонтальное растяжение показывается отдельно.

## Как проверить

1. Запустить программу.
2. Поставить большую глубину, например `100 м`.
3. Поставить длинную линию, например `160 м`.
4. Выполнить расчёт.
5. Открыть `2D-схема`.
6. Проверить, что якорь находится около дна, а не в середине воды.
7. Проверить, что на схеме есть подпись `X/Z масштабы разные`.
