# BuoyCalc Windows v0.24.4

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.24.4 исправляет граничные условия 2D-схемы: якорь всегда рисуется на дне, а буй рисуется на поверхности только тогда, когда длина линии позволяет дойти до поверхности.

## Что изменилось в v0.24.4

1. обновлён `Mooring2DCanvas`;
2. 2D-схема больше не строит всю форму только относительно якоря;
3. введена простая граничная логика:

```text
если LineLength >= Depth:
  буй у поверхности
  якорь на дне

если LineLength < Depth:
  буй под водой
  якорь на дне
```

4. форма линии растягивается между выбранными граничными условиями;
5. если расчётный отчёт был создан старой логикой, 2D-схема всё равно корректирует отображение;
6. окно 2D-схемы обновлено до `v0.24.4`.

## Почему это важно

Для длинной линии, например:

```text
Depth = 100 м
LineLength = 160 м
```

буй не должен визуально уходить под воду только из-за того, что форма рисуется от якоря. При такой геометрии линия имеет достаточную длину, чтобы верхний конец оказался у поверхности.

Для короткой линии, например:

```text
Depth = 100 м
LineLength = 36 м
```

верхний узел остаётся под водой, а якорь остаётся на дне.

## Как проверить

1. Запустить программу.
2. Поставить глубину `100 м`.
3. Поставить линию `160 м`.
4. Выполнить расчёт заново.
5. Открыть `2D-схема`.
6. Проверить, что окно имеет бейдж `v0.24.4`.
7. Проверить, что буй находится у поверхности, а якорь — на дне.
8. Затем поставить короткую линию меньше глубины и проверить, что буй уходит под воду.
