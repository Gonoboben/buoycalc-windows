# BuoyCalc Windows v0.23

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.23 переводит 2D-схему с условной прямой линии на рассчитанную форму по узлам `X/Z`.

## Что изменилось в v0.23

1. обновлён `Mooring2DCanvas`;
2. 2D-схема теперь ищет в отчёте раздел `Расчётные узлы линии X/Z`;
3. если узлы найдены, линия рисуется как полилиния по рассчитанным координатам;
4. буй ставится в первый узел;
5. якорь ставится в последний узел;
6. промежуточные точки линии показываются маркерами;
7. рядом со схемой добавлена подпись `форма по X/Z узлам`;
8. если расчёт ещё не выполнен или узлов нет, остаётся старая условная схема.

## Что важно

Это ещё не финальный `moordyn`, но визуализация уже использует результат расчёта:

```text
профиль течения
↓
сегменты линии
↓
натяжения и углы
↓
узлы X/Z
↓
2D-полилиния по узлам
```

## Текущая трактовка узлов

```text
Узел 0 = верхний конец линии у буя
последний узел = нижний конец линии у якоря
Lсег у последнего узла = длина последнего сегмента линии
```

Высота якоря пока не задаётся отдельным параметром и не влияет на расчёт `Z`.

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Открыть `2D-схема`.
4. Проверить, что на схеме есть подпись `форма по X/Z узлам`.
5. Проверить, что линия больше не абсолютно прямая: она строится по узлам из отчёта.
6. Проверить, что если расчёт не выполнен, схема всё равно открывается в старом fallback-режиме.
