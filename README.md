# BuoyCalc Windows v0.37.1

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.37.1 добавляет GitHub Actions CI для автоматической проверки сборки проекта. Это технический слой контроля качества: он не меняет расчётное ядро, 2D-схему, PDF или отчёт.

CI отвечает на вопрос:

```text
собирается ли проект после изменения кода
```

CI не является синхронизацией и не обновляет локальную программу на компьютере пользователя.

## Что изменилось в v0.37.1

1. Добавлен workflow:

```text
.github/workflows/dotnet-build.yml
```

2. Workflow запускается при:
   - `push` в `main`;
   - `pull_request` в `main`;
   - ручном запуске через `workflow_dispatch`.
3. Сборка выполняется на:

```text
windows-latest
```

4. Используется .NET 8:

```text
dotnet-version: 8.0.x
```

5. Команды сборки:

```text
dotnet restore BuoyCalc.Windows.csproj
dotnet build BuoyCalc.Windows.csproj --configuration Release --no-restore
```

6. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.37.1 - build CI
```

## Почему CI нужен

Проект быстро растёт: меняются `Models`, `Services`, `ReportBuilder`, PDF-генератор, 2D-canvas и ViewModel. Одна ошибка в типе, namespace или параметрах конструктора может сломать сборку.

GitHub Actions должен автоматически показать:

```text
зелёная галочка -> сборка прошла
красный крестик -> сборка сломалась
```

## Что CI не делает

CI не проверяет инженерную правильность расчёта.

CI не заменяет проверку PDF-отчётов.

CI не запускает ручной сценарий в интерфейсе Avalonia.

CI не обновляет приложение локально.

Для локального обновления всё ещё нужно:

```text
Git -> Pull
Build -> Clean Solution
Build -> Rebuild Solution
```

## Что осталось от v0.37

В приложении остаётся 2D-сравнение:

```text
основная форма MooringShapeSolver
альтернативная форма с дискретными нагрузками
дискретные X/Z-точки элементов
```

2D-схема по-прежнему не придумывает координаты: она только отображает расчётные данные из сервисов и хранилищ.

## Как проверить CI

1. Открыть репозиторий GitHub.
2. Перейти во вкладку `Actions`.
3. Найти workflow:

```text
.NET Build
```

4. Открыть последний запуск.
5. Проверить job:

```text
Build BuoyCalc Windows
```

6. Если он зелёный — проект собирается в GitHub Actions.
7. Если красный — открыть лог и посмотреть, на каком шаге ошибка:
   - restore;
   - build;
   - C# compile error;
   - проблема с пакетом.
