# BuoyCalc Windows v0.24.3

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.24.3 добавляет защиту непосредственно в 2D-отрисовку: последний X/Z-узел всегда трактуется как якорь и отображается на линии дна. Остальная форма линии рисуется вверх относительно якоря.

## Что изменилось в v0.24.3

1. обновлён `Mooring2DCanvas`;
2. 2D-схема теперь явно ставит последний X/Z-узел на дно;
3. экранная координата узлов считается относительно якоря:

```text
экранный Z узла = Depth - (Z_якоря - Z_узла)
```

4. если отчёт был создан старой логикой, визуализация всё равно не должна подвешивать якорь в толще воды;
5. добавлена подпись `якорь закреплён на дне`;
6. окно 2D-схемы обновлено до `v0.24.3`.

## Что было исправлено в v0.24.2

v0.24.2 привязывает нижний граничный узел X/Z к проектной глубине. Якорь на 2D-схеме должен находиться на дне, потому что расчётная постановка начинается от якоря.

## Что осталось от v0.24.1

В 2D-схеме сохраняются разные масштабы:

```text
Z-масштаб = по глубине воды
X-масштаб = по горизонтальному диапазону линии
```

Это нужно, чтобы длинная линия помещалась по ширине, но вертикальная шкала оставалась привязана к глубине.


## Как проверить

1. Запустить программу.
2. Поставить глубину `100 м`.
3. Поставить линию `160 м`.
4. Выполнить расчёт заново.
5. Открыть `2D-схема`.
6. Проверить, что окно имеет бейдж `v0.24.3`.
7. Проверить, что якорь находится на дне.
8. Проверить, что на схеме есть подпись `якорь закреплён на дне`.
