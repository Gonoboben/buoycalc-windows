# BuoyCalc Windows v0.35

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.35 строит альтернативную форму X/Z по натяжениям с дискретными нагрузками. Это следующий шаг после v0.34: соединители и приборы уже повлияли на накопленные натяжения и углы, а теперь по этим углам строится сравнительная форма линии.

Основной solver пока не заменяется. Новая форма выводится в отчёт как отдельная сравнительная ведомость.

## Что изменилось в v0.35

1. Добавлен сервис `MooringDiscreteLoadShapeBuilder`.
2. Он берёт:
   - основную форму `MooringShapeResult`;
   - натяжения с дискретными нагрузками `MooringDiscreteLoadTensionResult`.
3. Для сегментов используются углы из натяжений с дискретными нагрузками.
4. Масштаб углов подбирается итерационно, чтобы альтернативная форма попала в проектную глубину якоря:

```text
|Zякоря - Depth| <= 0.01 м
```

5. В отчёте появился раздел:

```text
## Альтернативная форма X/Z с дискретными нагрузками
```

6. В итогах отчёта появились строки:
   - `Снос альтернативной формы с дискретными нагрузками`;
   - `Отличие альтернативной формы от основной`;
   - `Статус альтернативной формы`.
7. В разделе области учёта элементов появилась колонка:

```text
Альт. форма X/Z
```

8. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.35 - alt shape report
```

## Что показывает альтернативная форма

Альтернативная форма отвечает на вопрос:

```text
как изменится X/Z-геометрия, если углы взять не из старых сегментных натяжений, а из натяжений с дискретными нагрузками от приборов и соединителей
```

В отчёте сравниваются:

```text
X основной формы
Z основной формы
X альтернативной формы
Z альтернативной формы
ΔX
ΔZ
снос основной формы
снос альтернативной формы
max Δузла
```

## Что v0.35 ещё не делает

v0.35 пока не заменяет основной solver и не вставляет дискретные элементы как отдельные X/Z-узлы. Соединители и приборы влияют на альтернативную форму через натяжения и углы, но ещё не появляются как отдельные точки на линии.

Полная схема ещё не замкнута:

```text
форма -> s-позиции -> дискретные нагрузки -> натяжения -> углы -> новая форма -> новые силы -> следующая итерация
```

Поэтому `Полный баланс всей постановки` остаётся `WARNING`.

## Что осталось от v0.34

В отчёте остаётся раздел:

```text
## Натяжения линии с дискретными нагрузками по s
```

Он показывает, какой вес и какая горизонтальная сила дискретных элементов находится ниже каждого сечения линии.

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...`.
3. В `Итоги` проверить строки:
   - `Снос альтернативной формы с дискретными нагрузками`;
   - `Отличие альтернативной формы от основной`;
   - `Статус альтернативной формы`.
4. Найти раздел:

```text
## Альтернативная форма X/Z с дискретными нагрузками
```

5. Проверить, что таблица содержит:
   - `X дискр.`;
   - `Z дискр.`;
   - `Угол дискр.`;
   - `Угол формы`;
   - `X осн.`;
   - `Z осн.`;
   - `ΔX`;
   - `ΔZ`.
6. Убедиться, что `Полный баланс всей постановки` пока остаётся `WARNING`.
