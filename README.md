# BuoyCalc Windows v0.19.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.19.1 исправляет вылет при редактировании скорости течения в окне `Профиль течения по глубине`. Причина была в рекурсивном обновлении строки `Summary`: изменение скорости обновляло Summary, а изменение Summary снова запускало обновление Summary.

## Что изменилось в v0.19.1

1. в `CurrentProfilePointViewModel` добавлена защита от повторного обновления `Summary`;
2. редактирование U / V / W больше не должно приводить к вылету;
3. профиль течения по глубине остаётся активным;
4. сохранение/загрузка профиля не менялись;
5. расчёт по-прежнему использует переходную оценку `U_eff = max sqrt(U^2 + V^2)`.

## Что было добавлено в v0.19

1. модель точки профиля течения;
2. окно `Профиль течения по глубине`;
3. поля профиля:
   - глубина, м;
   - U East, м/с;
   - V North, м/с;
   - W Vertical, м/с;
   - плотность воды, кг/м³;
4. включение/отключение профиля;
5. сохранение профиля в JSON проекта;
6. восстановление профиля при загрузке проекта;
7. раздел `Профиль течения по глубине` в отчёте.

## Ограничение v0.19.1

Полноценной послойной интеграции сил по сегментам ещё нет. Профиль уже вводится, сохраняется и попадает в отчёт, но расчёт временно использует его как эффективную скорость:

```text
U_eff = max sqrt(U^2 + V^2)
```

Послойный расчёт по глубине будет следующим этапом.

## Как проверить

1. Запустить программу.
2. Нажать `Профиль течения...`.
3. Включить галочку `Использовать профиль течения`.
4. Изменить значения `U East`, `V North` или `W Vertical`.
5. Проверить, что программа не вылетает.
6. Выполнить расчёт.
7. Проверить в блоке `Результат`, что появилась строка `Течение расчётное`.
8. Проверить в отчёте раздел `Профиль течения по глубине`.
