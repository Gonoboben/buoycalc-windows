# BuoyCalc Windows v0.20

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.20 добавляет первый слой сегментной физики: линии постановки теперь разбиваются на расчётные сегменты, каждому сегменту назначается оценочная глубина, из профиля течения берётся локальная скорость, затем считается локальная сила сопротивления.

## Что изменилось в v0.20

1. добавлена модель `SegmentCalculationRow`;
2. каждый участок линии разбивается на сегменты примерно по 1 м;
3. каждому сегменту назначается:
   - начальная координата вдоль линии `s0`;
   - конечная координата `s1`;
   - длина сегмента;
   - оценочная глубина;
   - локальные компоненты течения `U / V / W`;
   - локальная плотность воды;
   - площадь проекции;
   - коэффициент сопротивления;
   - сила сопротивления;
4. если включён профиль течения, течение интерполируется по глубине;
5. если профиль отключён, используется старое одно значение `CurrentSpeed`;
6. суммарная сила течения по линии теперь считается как сумма сил по сегментам;
7. в Markdown-отчёт добавлен раздел `Расчётные сегменты линии`;
8. в проверках появилась строка с количеством расчётных сегментов и суммарной силой по ним.

## Что важно

Это ещё не полный `moordyn` и не окончательная форма линии. В v0.20 глубина сегмента назначается оценочно вдоль длины линии:

```text
z_segment ≈ s / L_total × Depth
```

То есть модель уже готовит основу для расчёта по слоям, но ещё не выполняет итерационный поиск формы равновесия.

## Логика после v0.20

```text
профиль течения по глубине
↓
сегменты линии
↓
локальное течение на глубине сегмента
↓
локальная сила сопротивления
↓
суммарная сила по линии
↓
следующий этап: натяжения и координаты X/Y/Z
```

## Как проверить

1. Запустить программу.
2. Открыть `Профиль течения...`.
3. Включить `Использовать профиль течения`.
4. Задать разные скорости на разных глубинах.
5. Выполнить расчёт.
6. В отчёте найти раздел `Расчётные сегменты линии`.
7. Проверить, что у сегментов разные глубины и, при разном профиле, разные локальные скорости.
8. Проверить в разделе `Проверки`, что указано количество сегментов и суммарная сила течения по линии.
