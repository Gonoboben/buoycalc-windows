# BuoyCalc Windows v0.11.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.11.1 исправляет читаемость таблицы расчёта. Markdown-таблица остаётся в отчёте для копирования, но в интерфейсе появилась отдельная таблица элементов с изменяемой шириной колонок и прокруткой.

## Что изменилось в v0.11.1

1. добавлен пакет `Avalonia.Controls.DataGrid`;
2. подключены стили `DataGrid`;
3. в `MainWindowViewModel` добавлена коллекция `ElementRows`;
4. после расчёта строки элементов выводятся в отдельную таблицу;
5. в правой колонке появился блок `Таблица элементов`;
6. колонки таблицы можно растягивать мышью;
7. таблица имеет горизонтальную и вертикальную прокрутку;
8. отчёт больше не переносит длинные строки таблицы — включена горизонтальная прокрутка.

## Как обновиться локально

```text
Git → Pull
```

Затем запустить проект заново. При первом запуске после v0.11.1 Visual Studio может восстановить новый NuGet-пакет `Avalonia.Controls.DataGrid`.

## Как проверить v0.11.1

1. Запустить программу.
2. Нажать `Рассчитать систему`.
3. В правой колонке найти новый блок `Таблица элементов`.
4. Проверить, что таблица не разваливается на переносах строк.
5. Потянуть мышью границу между заголовками колонок.
6. Проверить, что ширина колонки меняется.
7. Проверить горизонтальную прокрутку таблицы.
8. Проверить блок `Отчёт`: длинные строки должны прокручиваться горизонтально, а не ломать таблицу переносами.

## Что важно понимать

Это всё ещё предварительный расчёт. v0.11.1 не меняет физику расчёта, а улучшает читаемость и контроль результата.

Текущие упрощения:

1. линия считается по проецируемой площади `длина × диаметр`;
2. волновая составляющая пока прикладывается только к бую;
3. якорь пока считается через вес в воде, коэффициент удержания и грунтовый множитель;
4. динамика, усталость, рывки, провисание линии и контакт с грунтом пока не моделируются.
