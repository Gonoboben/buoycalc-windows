# BuoyCalc Windows v0.29

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.29 добавляет первый итерационный solver формы постановки. Это ещё не полный нелинейный solver равновесия всей системы, но важный шаг: `MooringShapeSolver` больше не возвращает постоянное `Converged = false`. Теперь форма имеет реальный критерий сходимости по якорной глубине.

## Что изменилось в v0.29

1. В `MooringShapeResult` добавлены поля:
   - `IterationCount`;
   - `ConvergenceResidualM`;
   - `AngleScale`;
   - `ConvergenceCriterion`.
2. `MooringShapeSolver` теперь выполняет итерационный подбор масштаба углов сегментов.
3. Углы сегментов берутся из `SegmentTensionAnalyzer`, то есть из накопленных сил линии.
4. Масштаб углов подбирается методом бисекции.
5. Критерий сходимости:

```text
|Zякоря - Depth| ≤ 0,01 м
```

6. В инженерной диагностике появились строки:
   - `Итерации solver формы`;
   - `Невязка сходимости формы`;
   - обновлённый `Статус сходимости формы`.
7. Версия в главном окне обновлена через единый `AppInfo`:

```text
v0.29 · iterative shape solver
```

## Что именно решает v0.29

v0.29 решает **геометрическое замыкание формы**:

```text
сегментные силы → углы сегментов → X/Z узлы → проверка глубины якоря
```

Если линия длиннее глубины, solver подбирает такой масштаб углов, чтобы нижний узел линии совпал с проектной глубиной. Если линия короче глубины, верхний узел будет находиться под водой, а буй получит состояние `Submerged`.

## Что v0.29 ещё не решает

v0.29 ещё не решает полную нелинейную задачу:

```text
форма линии ⇄ натяжения ⇄ силы ⇄ реакции ⇄ новое положение узлов
```

Поэтому строка:

```text
Полный баланс всей постановки
```

по-прежнему остаётся `WARNING`.

Это сделано специально: программа не должна выдавать статус `OK` для физики, которая ещё не замкнута в единую систему.

## Что осталось от v0.28.1

В текстовом отчёте остаётся раздел:

```text
## Векторная ведомость сил постановки
```

Он показывает вклад каждого силового члена в `ΣFx` и `ΣFz`:

```text
буй
соединители
приборы
линия
якорь
волновая добавка, если она есть
```

Итоговые строки ведомости:

```text
Сумма учтённых сил
Требуемая реакция для ΣF=0
```

## Что осталось от v0.27.1

Строки внутреннего контроля линии остаются информационными:

```text
ΣFx линии
ΣFz линии
Контроль накопления ΣFx линии
Контроль накопления ΣFz линии
```

Они не являются полной проверкой равновесия всей системы. Это внутренний контроль накопления сил текущей сегментной модели линии.

## Методология направлений

Приняты направления:

```text
+X = направление действия течения / горизонтальной нагрузки
+Z = вверх
```

Для каждого элемента из расчётного ядра используется уже существующая строка `ElementCalculationRow`:

```text
Fx = CurrentForceN
Fz = -WeightWaterKg × g
```

Это значит:

```text
положительный WeightWaterKg  → сила вниз
отрицательный WeightWaterKg → плавучесть вверх
```

## Что осталось от v0.25.1

Основной 2D-вид использует равный масштаб:

```text
xScale = zScale
```

То есть координаты отображаются без искусственного увеличения по горизонтали.

## Как проверить

1. Запустить программу.
2. Убедиться, что в правом верхнем углу написано:

```text
v0.29 · iterative shape solver
```

3. Выполнить расчёт.
4. Открыть `Отчёт текстом...`.
5. Найти раздел `Контроль инженерной модели`.
6. Проверить строки:
   - `Итерации solver формы`;
   - `Невязка сходимости формы`;
   - `Статус сходимости формы`.
7. Проверить, что `Статус сходимости формы` больше не обязан быть `Preliminary`: при нормальной геометрии он должен стать `Converged`.
8. Проверить, что `Полный баланс всей постановки` пока остаётся `WARNING`.

## Следующий шаг

Рекомендуемый v0.30:

1. связать геометрическую форму с векторным балансом;
2. пересчитывать реакции после каждой итерации формы;
3. учитывать изменение проекций сегментов в локальных силах;
4. заменить текущий подбор масштаба углов на итерацию:

```text
форма → локальные силы → натяжения → углы → новая форма
```

5. после этого постепенно переводить `Полный баланс всей постановки` из `WARNING` в реальную проверку сходимости.
