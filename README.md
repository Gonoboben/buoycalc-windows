# BuoyCalc Windows v0.24

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.24 исправляет важную ошибку визуализации и таблицы узлов: верхний узел линии больше не считается всегда находящимся на поверхности. Если линия короче глубины, верхний узел смещается ниже поверхности, и 2D-схема показывает буй под водой.

## Что изменилось в v0.24

1. обновлён `MooringNodeAnalyzer`;
2. узлы `X/Z` теперь привязаны к нижнему концу линии у якоря;
3. если вертикальная протяжённость линии меньше глубины, вся линия смещается вниз;
4. у узла `0` может появиться `Z > 0`;
5. 2D-схема автоматически рисует буй ниже поверхности, потому что буй стоит в первом X/Z-узле;
6. обновлён заголовок и бейдж окна 2D-схемы до `v0.24`.

## Почему это важно

Раньше схема делала неверное предположение:

```text
узел 0 всегда Z = 0
```

Это работало для поверхностного буя, но было неверно для подводного варианта. Теперь логика такая:

```text
если линия достаточно длинная:
  верхний узел может быть у поверхности

если линия короче глубины:
  верхний узел находится под водой
  буй рисуется под водой
```

## Текущая логика расчёта узлов

```text
сегменты линии
↓
углы сегментов
↓
вертикальная протяжённость линии
↓
сравнение с глубиной нижнего узла
↓
смещение верхнего узла ниже поверхности, если линия короткая
```

Это всё ещё предварительная модель до полноценного итерационного расчёта равновесия, но для 2D-визуализации она уже устраняет главную ошибку: подводный буй не должен рисоваться на поверхности.

## Как проверить

1. Запустить программу.
2. Поставить короткую линию, например верхний буйреп около `26 м` при глубине `50 м`.
3. Выполнить расчёт.
4. Открыть `Отчёт текстом...`.
5. В разделе `Расчётные узлы линии X/Z` проверить, что у узла `0` глубина `Z` больше нуля.
6. Открыть `2D-схема`.
7. Проверить, что буй нарисован под поверхностью воды, а не на поверхности.
