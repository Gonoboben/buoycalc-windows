# BuoyCalc Windows v0.37

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.37 добавляет 2D-сравнение основной и альтернативной формы. Это визуальный слой поверх уже рассчитанных данных: основная форма берётся из `MooringShapeResult`, альтернативная форма и дискретные X/Z-точки берутся из расчётного слоя v0.35–v0.36 через отдельное хранилище. UI не придумывает координаты.

Основной solver пока не заменяется. 2D-схема теперь показывает сравнение, но не объявляет альтернативную форму окончательным решением.

## Что изменилось в v0.37

1. Добавлено временное хранилище `MooringAlternativeShapeStore`.
2. `MooringAlternativeDiscreteNodeProjector` теперь передаёт в это хранилище:
   - альтернативную форму `MooringDiscreteLoadShapeResult`;
   - дискретные X/Z-узлы `MooringAlternativeDiscreteNodeResult`.
3. `Mooring2DCanvas` теперь отображает:
   - основную форму `MooringShapeSolver`;
   - альтернативную форму с дискретными нагрузками;
   - дискретные X/Z-точки приборов, соединителей и вертлюгов.
4. Обе формы рисуются в одном масштабе:

```text
xScale = zScale
```

5. Синяя линия — основная форма.
6. Оранжевая линия — альтернативная форма.
7. Оранжевые точки — дискретные X/Z-узлы альтернативной формы.
8. Если альтернативная форма имеет предупреждение по `angleScale`, 2D-схема показывает это как `WARNING`.
9. Отображаемая версия окон и `AppInfo` обновлены:

```text
v0.37 - 2D comparison
```

## Что важно методически

2D-схема в v0.37 не парсит альтернативную форму из Markdown-отчёта. Координаты идут из расчётных объектов и временного хранилища:

```text
MooringDiscreteLoadShapeBuilder
        ↓
MooringAlternativeDiscreteNodeProjector
        ↓
MooringAlternativeShapeStore
        ↓
Mooring2DCanvas
```

Это соответствует правилу проекта: визуализация только отображает расчётную модель, а не создаёт свою физику.

## Что v0.37 ещё не делает

v0.37 не заменяет основной solver и не переносит альтернативную форму в итоговый баланс. Это режим сравнения:

```text
основная форма ≠ альтернативная форма
```

Альтернативная форма показывает влияние дискретных нагрузок, но пока не является финальным итерационным решением.

Полный баланс всей постановки остаётся `WARNING`.

## Что осталось от v0.36

В отчёте остаётся раздел:

```text
## Дискретные X/Z-узлы альтернативной формы
```

Он нужен для численной проверки того, что отображается на 2D-схеме.

## Как проверить

1. Выполнить расчёт.
2. Открыть `Отчёт текстом...` и убедиться, что есть раздел:

```text
## Дискретные X/Z-узлы альтернативной формы
```

3. Нажать `2D-схема`.
4. Проверить, что на схеме есть:
   - синяя основная линия;
   - оранжевая альтернативная линия;
   - оранжевые точки дискретных элементов;
   - подпись `альтернативная форма: OK` или `альтернативная форма: WARNING`.
5. Если в отчёте `angleScale > 2.0`, на 2D должна быть подпись `альтернативная форма: WARNING`.
6. Убедиться, что масштаб не растянут по горизонтали: `X=Z`.
7. Убедиться, что `Полный баланс всей постановки` пока остаётся `WARNING`.

## Следующий шаг

Рекомендуемый v0.38:

1. добавить PDF-страницу сравнения форм;
2. отрисовать в PDF основную и альтернативную форму в одном масштабе;
3. нанести дискретные X/Z-точки на PDF-схему;
4. затем, после проверки нескольких PDF с разными параметрами, перейти к итерационному solver:

```text
форма -> силы -> натяжения -> дискретные узлы -> новая форма -> сходимость
```
