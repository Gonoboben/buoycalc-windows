# BuoyCalc Windows v0.11

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.11 — первый этап улучшения расчётного ядра. Теперь расчёт не только выдаёт общие итоги, но и формирует таблицу по каждому элементу последовательности.

## Что уже есть

1. условия среды;
2. выбор буя из библиотеки;
3. последовательность постановки;
4. линии из библиотеки;
5. соединители из библиотеки;
6. приборы / нагрузки из библиотеки;
7. выбор якоря из библиотеки;
8. расчёт плавучести;
9. расчёт сопротивления течению;
10. упрощённая волновая нагрузка;
11. натяжение;
12. удержание якоря;
13. проверка слабого звена;
14. индивидуальные строки расчёта по элементам;
15. расширенный текстовый отчёт;
16. сохранение проекта в JSON;
17. загрузка проекта из JSON;
18. отдельное окно `Библиотека элементов`.

## Что изменилось в v0.11

1. добавлен `ElementCalculationRow`;
2. расширен `CalculationResult`;
3. расчёт теперь считает по каждому элементу:
   - вес в воде;
   - проецируемую площадь;
   - Cd;
   - силу сопротивления течению;
   - MBL;
   - WLL;
   - запас;
   - статус;
4. расчёт определяет слабое звено цепочки;
5. отдельно показываются сила течения и волновая составляющая;
6. в отчёте появилась таблица элементов;
7. проверки стали понятнее: слабое звено, индивидуальные запасы, запас якоря.

## Как проверить v0.11

1. Запустить программу.
2. Собрать последовательность из линии, соединителя и прибора.
3. Нажать `Рассчитать систему`.
4. В блоке `Отчёт` найти раздел `Таблица элементов`.
5. Проверить, что по каждому элементу указаны:
   - вес в воде;
   - площадь;
   - сила сопротивления;
   - MBL;
   - WLL;
   - запас;
   - статус.
6. Изменить линию или соединитель на более слабый элемент.
7. Нажать расчёт снова и проверить, что слабое звено и запас изменились.

## Что важно понимать

Это всё ещё предварительный расчёт. v0.11 делает расчёт прозрачнее и полезнее для проверки состава постановки, но не заменяет инженерную верификацию.

Текущие упрощения:

1. линия считается по проецируемой площади `длина × диаметр`;
2. волновая составляющая пока прикладывается только к бую;
3. якорь пока считается через вес в воде, коэффициент удержания и грунтовый множитель;
4. динамика, усталость, рывки, провисание линии и контакт с грунтом пока не моделируются.
