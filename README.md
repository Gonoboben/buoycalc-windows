# BuoyCalc Windows v0.25.1

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.25.1 исправляет инженерную 2D-визуализацию: основной вид теперь использует одинаковый масштаб по `X` и `Z`.

## Что изменилось в v0.25.1

1. обновлён `Mooring2DCanvas`;
2. основной инженерный 2D-вид больше не растягивает горизонтальную ось;
3. теперь используется правило:

```text
xScale = zScale
```

4. при глубине `100 м` и сносе `≈2 м` горизонтальное смещение будет маленьким, как и должно быть физически;
5. подпись на схеме изменена на `масштаб X=Z, без увеличения по горизонтали`.

## Почему это важно

Раздельные масштабы полезны для просмотра малых отклонений, но они искажают инженерную картину. Если пользователь видит 2D-схему без специального режима увеличения, координаты должны отображаться в одном масштабе.

Пример:

```text
Depth = 100 м
Horizontal offset = 1.95 м
```

Горизонтальное смещение должно быть около `1.95 %` от вертикальной глубины, а не занимать половину окна.

## Что было добавлено в v0.25

1. добавлен `MooringShapeSolver`;
2. добавлен `MooringShapeResult`;
3. добавлены точки формы `MooringShapePoint`;
4. добавлено состояние буя `BuoyShapeState`;
5. добавлен `MooringShapeStore` для хранения последней рассчитанной формы;
6. `ReportBuilder` теперь строит форму через `MooringShapeSolver`;
7. 2D-схема сначала берёт форму из `MooringShapeStore`;
8. Markdown-парсинг оставлен только как аварийный fallback для старого состояния;
9. в отчёте появился раздел `Расчётная форма постановки X/Z`.

## Новый принцип

```text
расчётное ядро
↓
MooringShapeResult
↓
отчёт / 2D / PDF
```

Визуализация не должна быть источником инженерной логики. Она должна только отображать расчётные координаты.

## Что ещё остаётся переходным

`MooringShapeSolver` пока использует предварительную квазистатическую модель на основе уже существующих сегментов, сил, натяжений и углов. Полный итерационный расчёт равновесия ещё не включён.

Текущее поле:

```text
Converged = false
```

означает, что форма уже вынесена в инженерный объект, но полноценная сходимость ещё будет добавляться следующим этапом.

## Как обновиться локально


```text

Build → Rebuild Solution
```

## Как проверить

1. Запустить программу.
2. Выполнить расчёт.
3. Открыть `2D-схема`.
4. Проверить подпись `форма из MooringShapeSolver`.
5. Проверить подпись `масштаб X=Z, без увеличения по горизонтали`.
6. Для случая `Depth = 100 м`, `снос ≈2 м` линия должна быть почти вертикальной, а не растянутой по горизонтали.
