# BuoyCalc Windows v0.21

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.21 добавляет первый расчёт натяжений по сегментам линии. После v0.20 у нас уже были сегменты, глубины и локальные силы течения. Теперь для каждого сегмента оцениваются вес в воде, накопленная горизонтальная сила, накопленная вертикальная сила, натяжение и угол от вертикали.

## Что изменилось в v0.21

1. добавлен сервис `SegmentTensionAnalyzer`;
2. добавлена модель `SegmentTensionRow`;
3. расчёт натяжений выполняется снизу вверх: от якоря к бую;
4. для каждого сегмента считается:
   - вес в воде;
   - локальная сила течения;
   - накопленная горизонтальная сила;
   - накопленная вертикальная сила;
   - оценочное натяжение;
   - угол от вертикали;
5. в разделе `Итоги` появился максимум натяжения по сегментной оценке;
6. в отчёт добавлен раздел `Натяжения по сегментам линии`.

## Что важно

Это ещё не полный `moordyn`. Сейчас натяжение считается по накопленным силам снизу вверх, но координаты линии пока не итерационные. То есть это переходная физическая модель:

```text
сегменты линии
↓
локальные силы
↓
накопленные силы снизу вверх
↓
оценочное натяжение
↓
следующий этап: координаты узлов и форма линии
```

## Как проверить

1. Запустить программу.
2. Открыть `Профиль течения...`.
3. Включить `Использовать профиль течения`.
4. Выполнить расчёт.
5. В отчёте найти раздел `Расчётные сегменты линии`.
6. Ниже найти раздел `Натяжения по сегментам линии`.
7. Проверить, что по сегментам есть:
   - вес в воде;
   - Fтек;
   - ΣFгор;
   - ΣFверт;
   - T, кН;
   - угол от вертикали.
8. В разделе `Итоги` проверить строку `Макс. натяжение по сегментной оценке`.
