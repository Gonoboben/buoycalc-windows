# BuoyCalc Windows v0.46.2

Windows-ветка проекта BuoyCalc на C# + Avalonia.

## Статус

v0.46.2 разделяет два типа отчётов:

```text
Пользовательский PDF-отчёт — краткий итог, схема, цепочка и таблица элементов.
Полный технический отчёт — отдельный текстовый отчёт в приложении со всеми диагностическими таблицами.
```

Расчётная физика, solver, 2D-координаты и форма с дискретными элементами не изменены.

## Что изменено

1. Обновлён файл:

```text
Services/PdfReportBuilder.cs
```

2. PDF больше не включает полный технический отчёт.
3. PDF теперь содержит только пользовательские разделы:

```text
Краткий итог
Схема постановки
Текстовая цепочка
Таблица элементов
Короткое примечание
```

4. Полный отчёт остаётся доступен отдельно через окно приложения:

```text
Полный отчёт
```

5. В пользовательском PDF больше не должны появляться длинные solver-таблицы:

```text
Позиционная модель последовательности по s
Область учёта элементов
Расчётные сегменты линии
Расчётная форма постановки X/Z
Проекции формы X/Z
Shape-based силы
Shape-based натяжения
Натяжения с дискретными нагрузками
Альтернативная форма X/Z
Итерационный solver
Автопроверки
```

6. Версия приложения:

```text
v0.46.2 - user and full reports
```

## Что не изменилось

1. Физика расчёта не изменилась.
2. Solver не изменился.
3. Gate не удалён.
4. Fallback-форма не удалена из приложения.
5. PDF-схема берёт координаты из расчётного слоя дискретных нагрузок.
6. Release build workflow остаётся тем же.

## Как проверить

1. Выполнить расчёт.
2. Экспортировать PDF.
3. Проверить, что PDF занимает несколько страниц, а не десятки страниц.
4. Проверить, что в PDF есть:

```text
пользовательский отчёт
Краткий итог
Схема постановки
Таблица элементов
```

5. Проверить, что полного технического отчёта внутри PDF нет.
6. Открыть отдельно окно `Полный отчёт` и убедиться, что техническая диагностика осталась там.
7. Проверить CI:

```text
BuoyCalc Windows Build: success
```
