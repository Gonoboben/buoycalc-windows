# BuoyCalc Windows v0.10

Windows-ветка проекта BuoyCalc на **C# + Avalonia**.

Это портирование расчётной логики и интерфейса в desktop-приложение:

```text
SwiftUI iPhone prototype
↓
C# / Avalonia Windows desktop prototype
```

## Статус

v0.10 добавляет библиотеку приборов / нагрузок. Теперь элемент `Payload` в последовательности выбирается из библиотеки, а масса, объём, площадь и Cd подставляются автоматически.

## Что уже есть

1. условия среды;
2. выбор буя из библиотеки;
3. последовательность постановки;
4. линии из библиотеки;
5. соединители из библиотеки;
6. приборы / нагрузки из библиотеки;
7. выбор якоря из библиотеки;
8. расчёт плавучести;
9. расчёт сопротивления течению;
10. упрощённая волновая нагрузка;
11. натяжение;
12. удержание якоря;
13. проверки системы;
14. текстовый отчёт;
15. сохранение проекта в JSON;
16. загрузка проекта из JSON;
17. отдельное окно `Библиотека элементов`.

## Что изменилось в v0.10

1. добавлена модель `PayloadLibraryItem`;
2. добавлен сервис `PayloadLibraryStorage`;
3. в окне `Библиотека элементов` появилась вкладка `Приборы`;
4. можно создать пользовательский прибор;
5. можно сохранить прибор;
6. можно удалить пользовательский прибор;
7. встроенные приборы удалить нельзя;
8. пользовательские приборы сохраняются в JSON;
9. в карточке `Payload` появился список `Пресет прибора`;
10. при выборе прибора автоматически подставляются масса, объём, площадь и Cd;
11. в проекте сохраняется выбранный ID прибора.

## Как проверить v0.10

1. В верхней панели главного окна нажать `Библиотека элементов...`.
2. Открыть вкладку `Приборы`.
3. Выбрать встроенный прибор и посмотреть параметры.
4. Нажать `Новый прибор`.
5. Заполнить название, тип, массу, объём, площадь и Cd.
6. Нажать `Сохранить`.
7. Закрыть окно библиотеки.
8. В главном окне нажать `+ прибор`.
9. В карточке прибора открыть список `Пресет прибора`.
10. Выбрать созданный прибор.
11. Проверить, что масса, объём, площадь и Cd подставились.
12. Нажать `Рассчитать систему`.

## Когда улучшаем расчёт

Следующий крупный этап — **v0.11: улучшение расчётного ядра**.

Теперь библиотеки основных элементов уже есть:

```text
буи
линии
соединители
приборы
якоря
```

Поэтому в v0.11 можно переходить к инженерной части:

1. проверка слабого звена по всей цепочке;
2. запас по MBL для каждой линии и каждого соединителя;
3. вес в воде по каждому элементу;
4. вклад каждого элемента в сопротивление течению;
5. отдельная логика якоря по типу;
6. поправка удержания на грунт;
7. более подробные предупреждения;
8. таблица вкладов элементов в отчёте.

## Следующий шаг

2. расширить `CalculationResult`;
3. добавить таблицу слабых мест и вкладов элементов в отчёт.
